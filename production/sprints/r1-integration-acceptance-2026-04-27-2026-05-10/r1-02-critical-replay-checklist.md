# R1 关键链路回放清单（Critical Replay Checklist）

- 阶段：R1 集成验收（2026-04-27 ~ 2026-05-10）
- 基线分支：`release/baseline-2026Q2`
- 关联文档：`r1-01-acceptance-template.md`
- 目标：确保跨系统主链路可稳定回放、可留证据、可复验签收

## 使用规则
1. 每条链路至少执行 1 次完整回放；高风险链路（RL-01/03/10）至少执行 2 次。
2. `状态` 仅填：`PASS / FAIL / BLOCKED`。
3. 任何 `FAIL/BLOCKED` 必须绑定缺陷ID，并在修复后补填 `复验结果`。
4. 证据统一落在 `production/evidence/r1/` 下，按 `链路ID/日期/` 建目录。
5. 涉及存档的链路必须包含“保存 -> 退出 -> 重进 -> 读取 -> 继续推进”闭环。
