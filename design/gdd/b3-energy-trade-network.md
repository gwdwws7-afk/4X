# B3 能源贸易网络 (Energy Trade Network)

> **状态**: Designed
> **作者**: GDC Framework / 游戏设计师
> **最后更新**: 2026-03-31
> **实现支柱**: P0（不对称博弈核心场景）, P1（经济结算基础）

## Overview 概述

能源贸易网络是连接B1（星币-能源金融）和H2（战略节点）的桥梁，定义能源如何在各节点之间流动、贸易路线如何被切断或绕行、以及路线控制带来的收入加成/惩罚。

本系统的核心问题是：能源从瓦希德高原产出后，如何到达买家手中？哪些路线可以被封锁？哪些替代路线可以绕过封锁？

## Player Fantasy 玩家幻想

玩家感受到"每一条贸易路线都是一条生命线"的紧迫感。

火海咽喉是瓦希德的命脉——但一旦被封锁，能源就卖不出去。东土商路是备用的生命线——但要穿过北境通道，而北境共主随时可能涨价。红海航线被金帐合众控制，只能在"允许"的情况下使用。

每一次军事行动、外交谈判、节点争夺，都在影响贸易网络的可用性和效率。控制关键节点就是控制钱袋子。

## Detailed Design 详细设计

### Core Rules 核心规则

#### 贸易网络架构

```
能源贸易网络拓扑
│
├── 瓦希德能源产地（起点）
│   ├── 瓦希德高原（+8火油/回合）
│   └── 基尔库克（+6火油/回合，归属中立时）
│
├── 贸易路线
│   ├── 主路线：瓦希德高原 → 火海咽喉 → [红海/地中海]
│   ├── 备用路线A：瓦希德高原 → 北境通道 → 北境共主
│   ├── 备用路线B：瓦希德高原 → 两河联邦 → 东土商路
│   └── 灰色路线：任何节点 → 中立转运港（效率最低）
│
└── 目的地（买家）
    ├── 东土商盟（愿意用商盟券结算）
    ├── 北境共主（愿意以物易物结算）
    ├── 金帐合众（正常星币结算，但可能封锁）
    └── 其他国家（通过中立港转运）
```

#### 关键贸易路线定义

| 路线ID | 路线名称 | 节点序列 | 长度 | 控制势力 | 重要性 |
|--------|---------|---------|------|---------|--------|
| route_persian_gulf | **火海航线** | 瓦希德高原→火海咽喉→贝鲁特 | 2跳 | 瓦希德(咽喉) | 极高（主要出口通道） |
| route_northern | **北境商路** | 瓦希德高原→北境通道→东土商路 | 2跳 | 北境共主(通道) | 高（替代出口） |
| route_tigris | **两河走廊** | 基尔库克→两河联邦→东土商路 | 2跳 | 中立 | 中（能源过境） |
| route_red_sea | **红海航线** | 吉布提→红海咽喉→贝鲁特 | 2跳 | 金帐合众 | 高（金帐合众生命线） |

#### 路线状态

| 路线状态 | 描述 | 贸易效率 | 触发条件 |
|---------|------|---------|---------|
| **畅通** | 路线完全可用 | 100% | 所有节点均在友方/中立控制 |
| **受压** | 路线存在风险 | 70% | 任一节点处于争夺中 |
| **中断** | 路线不可用 | 0% | 任一节点被敌方控制 |

**注**：本系统采用全有全无机制，不支持单向通行。特殊单向效果（如"封锁出口但允许进口"）由D1军事行动系统作为附加效果实现。|
| **替代** | 绕过中断段 | 60% | 主路线中断后启用 |

#### 路线控制权计算

```
路线控制权 = Σ(各节点控制比例 × 节点权重)

节点权重：
- 起点/终点: 0.3
- 中间咽喉: 0.4

示例：
路线：瓦希德高原(100%瓦希德) → 火海咽喉(60%瓦希德/40%争夺中) → 贝鲁特(100%中立)
路线控制权 = 1.0×0.3 + 0.6×0.4 + 1.0×0.3 = 0.84（84%）
贸易效率 = 84%（畅通偏下）
```

### 贸易结算流程

#### 回合内贸易结算顺序

```
回合结算 - B3 能源贸易
│
├── 1. 读取各能源节点产出（H2提供）
│   └── 瓦希德高原: +8/基尔库克: +6
│
├── 2. 评估各贸易路线状态
│   ├── 检查节点归属（H2提供）
│   ├── 检查节点争夺状态（H2提供）
│   └── 计算各路线效率
│
├── 3. 计算可用贸易量
│   ├── 能源总产出 = Σ(各能源节点产出)
│   ├── 国内消耗 = 能源总产出 × 0.3
│   ├── 可出口量 = 能源总产出 × 0.7
│   └── 按路线分配（根据畅通情况和买家需求）
│
├── 4. 计算各买家贸易量
│   ├── 东土商盟（商盟券渠道）
│   ├── 北境共主（本币结算）
│   ├── 金帐合众（星币结算）
│   └── 其他（灰色市场）
│
├── 5. 汇总贸易数据
│   └── 输出至B1进行货币结算
```

### 路线效率与贸易量

#### 路线效率计算

```
单跳贸易效率 = 
    节点A效率 × 节点B效率（两者平均）

路线总效率 = Π(各跳效率) × 路线修正

路线修正：
- 路线是"主要出口通道": ×1.1（加成）
- 路线被用作"替代路线": ×0.9（轻微惩罚）
- 路线长度 > 2跳: 每增加1跳 ×0.9
```

#### 替代路线触发

当主路线中断时，系统自动寻找替代路线：

```
主路线中断检测：
if 火海航线.状态 == 中断:
    尝试路线2：北境商路
    if 北境商路.状态 == 畅通:
        启用北境商路（效率60%）
    else:
        尝试路线3：两河走廊
        if 两河走廊.状态 == 畅通:
            启用两河走廊（效率55%）
        else:
            启用灰色路线（效率40%）

默认模式：系统自动选择最高效率路线
手动模式：玩家可指定某条路线（UI显示所有可用路线）
```

#### 贸易量分配优先级

当能源可供出口量有限时，按以下优先级分配：

| 优先级 | 买家 | 结算方式 | 分配逻辑 |
|--------|------|---------|---------|
| 1 | 国内消耗 | — | 优先保障30%国内消耗 |
| 2 | 北境共主 | 以物易物 | 外交关系保险，武器换能源 |
| 3 | 东土商盟 | 商盟券 | 绕过星币结算的生命线 |
| 4 | 金帐合众 | 星币 | 正常出口，但可能被封锁 |
| 5 | 其他国家 | 灰色市场 | 最后手段，数量有限 |

### 路线控制与争夺

#### 节点归属对路线的影响

| 节点归属变化 | 对路线的影响 |
|------------|-------------|
| 瓦希德失去火海咽喉控制 | 火海航线立即中断（0%） |
| 瓦希德控制贝鲁特 | 火海航线效率+20%（终点加成） |
| 中立国控制两河联邦 | 两河走廊可用（需谈判） |
| 北境共主封锁北境通道 | 北境商路立即中断 |

#### 路线争夺联动

```
当D1军事行动导致节点状态变化时：
1. H2通知B3节点归属/状态变化
2. B3重新计算受影响路线的效率
3. B3通知B1可用贸易量变化
4. B1重新计算能源出口收入
```

### 贸易路线可视化（H1联动）

| 路线状态 | H1视觉表现 |
|---------|-----------|
| 畅通 | 淡金色虚线，流动动画 |
| 受压 | 橙色虚线，流动变慢 |
| 中断 | 红色×标记，无流动 |
| 替代激活 | 绿色虚线，标注"替代路线" |

## Formulas 公式

### 核心计算

#### 公式1：路线效率
```
单跳效率 = (起点节点效率 + 终点节点效率) / 2

节点效率 = 
    友方控制: 1.0
    中立控制: 0.8
    争夺中（初期）: 0.6
    争夺中（深度）: 0.2
    敌方控制: 0.0

路线总效率 = Π(各跳效率) × 长度修正 × 通道加成
长度修正 = 0.9^(路线跳数-1)
```

#### 公式2：可用贸易量
```
可用贸易量 = 能源总产出 × 出口比例 × 路线总效率

能源总产出 = Σ(各能源节点基础产出)
出口比例 = 0.7（固定）

示例：
瓦希德高原+8，基尔库克（中立）+6
能源总产出 = 14
可用贸易量 = 14 × 0.7 × 0.8（路线效率）= 7.84单位
```

#### 公式3：贸易收入加成
```
路线收入加成 = 1 + Σ(加成来源)

加成来源：
- 控制路线终点节点: +0.2
- 控制路线全部节点: +0.3（额外）
- 路线为"主要出口通道": +0.1
- 东土商盟友好关系≥友善: +0.1
```

#### 公式4：替代路线效率
```
替代路线效率 = 主路线中断时系统选择效率 × 0.9（替代惩罚）

替代路线选择顺序：
1. 北境商路（若北境共主同意）: 60%
2. 两河走廊（若两河联邦友好）: 55%
3. 灰色转运（任何情况可用）: 40%

若多条替代路线可用，取最高效率路线
```

#### 公式5：贸易枢纽加成
```
贸易枢纽金叶产出 = 基础产出 × (1 + 路线加成)

路线加成 = Σ(每条活跃连接路线 × 0.2)
活跃连接 = 双方节点均无迷雾且无深度争夺

示例：
贝鲁特连接：火海咽喉(畅通)、大马士革(争夺中)
活跃路线 = 1（仅火海咽喉）
路线加成 = 1 × 0.2 = 0.2
贝鲁特金叶产出 = 基础产出 × 1.2
```

## Edge Cases 边界情况

| 边界情况 | 处理方式 |
|---------|---------|
| 所有路线均中断 | 能源完全无法出口，只能国内消耗；提示玩家"贸易网络崩溃" |
| 能源节点被切断但仍有产出 | 产出囤积（累积型），下回合优先消耗库存 |
| 北境共主同时是路线控制方和买方 | 北境本币结算优先（绕过封锁效率最高） |
| 多条路线同时中断 | 自动切换最高效率替代路线 |
| 贸易量大于买方需求 | 按优先级分配，超出部分转为库存或灰色市场 |
| 节点在回合中途切换归属 | 立即重新计算路线效率，本回合按新旧效率比例结算 |

## Data Interfaces 数据接口

### B3向其他系统提供的接口

```typescript
interface TradeNetworkSystem {
    // 获取所有贸易路线状态
    getAllRouteStatuses(): RouteStatus[]
    
    // 获取某路线的当前效率
    getRouteEfficiency(routeId: string): number
    
    // 获取可用贸易量（按结算方式分类）
    getAvailableTradeAmount(): TradeAllocation
    
    // 获取替代路线推荐
    getAlternativeRoute(mainRouteId: string): AlternativeRoute[]
    
    // 获取路线贸易收入加成
    getRouteIncomeBonus(): RouteBonus[]
}

interface RouteStatus {
    routeId: string
    routeName: string
    nodes: string[]
    efficiency: number
    status: 'open' | 'pressured' | 'blocked' | 'alternative'
    controllingFaction: FactionId
    incomeBonus: number
}

interface TradeAllocation {
    totalAvailable: number
    byCurrency: {
        goldLeaves: number      // 星币结算
        tradeNotes: number      // 商盟券结算
        northCoins: number      // 北境本币结算
        barter: number          // 以物易物
        greyMarket: number      // 灰色市场
    }
    byRoute: {
        routeId: string
        amount: number
        efficiency: number
    }
}
```

## Tuning Knobs 调参旋钮

| 旋钮 | 默认值 | 可调范围 | 调整后果 |
|------|--------|---------|---------|
| 出口比例 | 70% | 50%-80% | 影响国内消耗vs出口分配 |
| 咽喉要道权重 | 0.4 | 0.3-0.5 | 影响咽喉控制对路线的重要性 |
| 路线主要通道加成 | 10% | 5%-15% | 影响主要航线价值 |
| 替代路线基础效率 | 55-60% | 40%-70% | 影响绕过封锁的吸引力 |
| 替代路线惩罚 | 10% | 5%-15% | 影响替代路线vs主路线选择 |
| 路线长度修正系数 | 0.9/跳 | 0.85-0.95 | 影响长途贸易价值 |
| 贸易枢纽加成系数 | 0.2/活跃路线 | 0.15-0.25 | 影响控制多个节点的价值 |
| 替代路线默认选择 | 自动最优 | 自动/手动 | 自动省操作；手动提供策略自主权 |
| 争夺中节点效率（初期） | 60% | 50%-70% | 影响争夺对路线的影响 |
| 争夺中节点效率（深度） | 20% | 10%-30% | 影响深度争夺的惩罚程度 |

**默认值已确定，待Alpha阶段平衡测试验证。**
