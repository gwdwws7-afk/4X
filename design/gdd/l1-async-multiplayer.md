# L1 异步多人系统 (Asynchronous Multiplayer System)

> **状态**: Draft
> **作者**: GDC Framework / 游戏设计师
> **最后更新**: 2026-03-31
> **实现支柱**: P4（多人游戏支持）

## Overview 概述

异步多人系统支持玩家之间异步对战——玩家可以发送自己的回合数据给对手，对手在方便的时候继续游戏。

## Detailed Design 详细设计

### Core Rules 核心规则

#### 异步对战流程

```
1. 玩家A创建异步房间
2. 玩家A完成自己的回合，发送数据
3. 系统通知玩家B
4. 玩家B在方便时登录，完成回合，发送数据
5. 系统通知玩家A
...
6. 直到某方胜利或放弃
```

#### 房间状态

| 状态 | 说明 |
|------|------|
| **等待中** | 等待对手加入 |
| **瓦希德回合** | 玩家A行动中 |
| **黄金领回合** | 玩家B行动中 |
| **已完成** | 游戏结束 |
| **已放弃** | 某方放弃 |

#### 回合数据同步

```
回合数据 = {
    回合数: number,
    瓦希德行动: Action[],
    黄金领行动: Action[],
    当前状态: GameState
}
```

### 系统架构

```
异步多人架构：

玩家A客户端 ←→ 游戏服务器 ←→ 玩家B客户端
                    ↓
              房间管理
                    ↓
              数据存储
```
