# A3 资源系统 (Resource System)

> **状态**: 设计中
> **作者**: GDC Framework / 游戏设计师
> **最后更新**: 2026-03-31
> **实现支柱**: P0（不对称博弈依赖资源稀缺性）, P1（代价系统依赖资源消耗）

## Overview 概述

资源系统是游戏的经济基础，定义了7种资源的产生、消耗、囤积和上限管理规则。它与回合主循环（A1）共同构成游戏的底层框架，所有其他系统（经济/外交/军事/派系）都在资源系统之上运转。

资源分为三种类型：消耗型（用完即止）、累积型（可囤可积）、比率型（影响效率系数）。玩家通过行动获得、消耗和管理这些资源，资源的稀缺性是P0不对称博弈的核心支撑。

## Player Fantasy 玩家幻想

玩家感受到"资源永远不够用"的窒息感。

在每一个回合中，玩家会面临这样的处境：能源出口收入因为封锁而锐减，但国内的社稷值又在因为国际压力而摇摇欲坠；手上的战械只够支持一次大规模军事行动，但外交谈判需要消耗灰烬志来提升谈判筹码，而灰烬志又不足以同时支撑两线操作。

这种稀缺感不是设计的缺陷，而是核心体验。玩家在"什么都要省着用"中做出取舍决策，在"拆东墙补西墙"中感受抵抗轴心的悲壮感。当玩家成功用有限的资源同时支撑了军事行动、维持了内部稳定、还在外交上取得了突破，那种"四两拨千斤"的成就感就是最强的正反馈。

**设计目标**：玩家在每回合结束时感到"我手上的资源刚好够用，但不能再多了"——而不是"资源充沛想怎么花就怎么花"。

## Detailed Design 详细设计

### Core Rules 核心规则

#### 资源三类七种

| 资源 | 虚构名 | 类型 | 基础值 | 初始上限 | 说明 |
|------|--------|------|--------|---------|------|
| 军事装备 | 战械 | 消耗型 | 10 | 20（战略储备上限） | 军事行动主要消耗，可囤积；无仓库容量上限，但超出的部分自动进入战略储备池（上限20） |
| 能源 | 火油 | 累积型 | 20 | 100 | 核心收入来源，封锁下出口受阻 |
| 星币 | 金叶 | 累积型 | 5 | 80 | 国际结算货币，被封锁后获取困难 |
| 友好货币 | 商盟券 | 累积型 | 0 | 60 | 绕过星币结算的替代货币 |
| 内部稳定 | 社稷值 | 比率型 | 70 | 0-100 | 决定国家效率折扣（100%=全力运转，50%=半速） |
| 意识形态 | 灰烬志 | 比率型 | 30 | 0-100 | 影响意识形态扩散效率和抵抗叙事感染率 |
| 国际地位 | 朝贡序 | 比率型 | 20 | 0-100 | 影响外交筹码、联合国话语权 |

#### 三类资源的行为规则

**消耗型（战械）**
- 使用后减少，不补充就会耗尽
- 可通过以下方式获取：北境共主武器转让、战场缴获、自主生产
- 有仓库容量上限（影响囤积上限）
- 消耗型资源不存在衰减

**累积型（火油/金叶/商盟券）**
- 可以囤积，上限可通过外交、科技、建筑提升
- 基础上限确保游戏平衡
- 无被动衰减，但有自然消耗（如军事行动消耗）
- 可进行相互转化（金叶可兑换商盟券，汇率由B4汇率系统决定）

**比率型（社稷值/灰烬志/朝贡序）**
- 无衰减（靠主动行动维持）
- 影响其他系统的效率系数
- 执行相关行动时才会变化
- 最高100，最低0，不可为负数

#### 比率型资源的效果映射

| 比率型资源 | 效果 | 量化公式 |
|-----------|------|---------|
| 社稷值 | 100%=正常效率；每下降10%，所有资源产出减少5% | `实际效率 = 基础效率 × (社稷值 / 100)` |
| 灰烬志 | 决定意识形态行动的成功率和感染率 | `意识形态行动基础成功率 = 30% + (灰烬志 × 0.5%)`，范围：30%-80% |
| 朝贡序 | 影响外交谈判成功率和联合国决议投票权重 | `外交谈判加成 = 朝贡序 / 200`，投票权重 = 基础权重 × (1 + 朝贡序/100) |

#### 资源获取基本规则

- 所有资源通过行动获取，不存在"自动收入"
- 回合结算时，无任何被动资源产出（这是4X游戏中不寻常的设计，强化稀缺感）
- 能源（火油）每回合通过国内消费和国际出口获得收入，但封锁严重影响出口收入

### States and Transitions 状态与转换

资源系统本身不维护独立状态机。它维护的是7种资源在游戏进程中的数值状态。

资源状态变化由以下事件触发：
1. 玩家执行消耗/获取资源的行动 → 资源实时变化
2. 回合结算 → 触发经济结算（B5经济结算系统）
3. 外交事件 → 影响朝贡序
4. 军事事件 → 影响战械存量和社稷值
5. 意识形态事件 → 影响灰烬志

### Interactions with Other Systems 与其他系统的交互

#### 上游依赖

本系统无上游依赖（Foundation层）。

#### 下游依赖

| 系统 | 依赖性质 | 依赖内容 |
|------|---------|---------|

| B5 经济结算系统 | 硬依赖 | 回合结算时对7种资源进行净值计算 |
| D1 军事行动系统 | 硬依赖 | 战械作为军事行动的消耗资源 |
| D2 军事-政治联动 | 硬依赖 | 社稷值作为军事行动的代价之一 |
| C2 外交协议系统 | 软依赖 | 外交协议消耗/产出资源 |
| C3 意识形态输出系统 | 硬依赖 | 灰烬志作为意识形态行动的核心资源 |
| E 内部政治系统 | 硬依赖 | 社稷值是派系满意度的核心决定因素 |
| K UI框架 | 硬依赖 | 7种资源的数值显示 |

#### 数据接口

本系统向其他系统提供：
- `getResource(type, quantity)` — 查询某类资源当前值
- `spendResource(type, quantity)` — 消耗指定数量资源
- `addResource(type, quantity)` — 增加指定数量资源
- `checkResourceLimit(type)` — 查询某类资源的当前上限

其他系统不得直接修改资源数值，必须通过上述接口。

## Formulas 公式

### 核心计算

#### 公式1：比率型资源效率折扣
```
实际效率 = 基础效率 × (社稷值 / 100)
当社稷值 = 70时：实际效率 = 基础效率 × 0.7
```

#### 公式2：累积型资源动态上限
```
资源上限 = 基础上限 + 上限加成
基础上限：火油=100，金叶=80，商盟券=60
上限加成来源：外交协议+5，科技投资+10，建筑设施+15
```

#### 公式3：能源收入计算
```
能源收入 = (国内能源产量 × 国内消耗系数) + (出口能源量 × 出口价格 × (1-封锁折扣))
封锁折扣：0%（无封锁）到 90%（全面封锁）
```

#### 公式4：资源变化通知阈值
```
当任意资源变化超过 ±5 时，触发UI通知
当比率型资源变化超过 ±10 时，触发重要事件提示
```

## Edge Cases 边界情况

| 边界情况 | 处理方式 |
|---------|---------|
| 资源达到上限 | 超出部分自动存入"战略储备"（不受上限限制的特殊池，用于紧急情况） |
| 资源归零 | 消耗型（战械）归零后无法执行相关行动；比率型归零触发国内危机事件 |
| 比率型资源超过100 | 自动锁定在100，超出部分溢出 |
| 资源不足仍尝试行动 | 行动按钮灰色不可用，显示"资源不足" |
| 存档损坏 | 存档系统（A4）负责处理，资源系统提供所有资源快照 |

## Tuning Knobs 调参旋钮

| 旋钮 | 默认值 | 可调范围 | 调整后果 |
|------|--------|---------|---------|
| 社稷值效率折扣系数 | 0.05 | 0.01-0.1 | 过高→稳定度敏感；过低→稳定度无意义 |
| 能源基础上限 | 100 | 50-200 | 过高→囤积无感；过低→经济压力过大 |
| 金叶基础上限 | 80 | 40-150 | 过高→封锁惩罚弱；过低→金融工具稀缺 |
| 商盟券基础上限 | 60 | 30-100 | 过高→替代货币过于强大；过低→绕过封锁困难 |
| 战械初始值 | 10 | 5-20 | 过高→军事过度强势；过低→早期军事受限 |
| 灰烬志初始值 | 30 | 10-50 | 过高→意识形态免费扩散；过低→需要大量投入才能起步 |
| 朝贡序初始值 | 20 | 10-40 | 过高→外交过于容易；过低→外交筹码稀缺 |
| 战略储备池上限 | 20 | 10-30 | 过高→绕过上限机制；过低→紧急情况无缓冲 |

**默认值已确定。**

## Acceptance Criteria 验收标准

### 基本资源操作
- [ ] 7种资源在UI上正确显示数值
- [ ] 消耗型资源（战械）执行军事行动后正确减少
- [ ] 累积型资源（火油/金叶/商盟券）可正常囤积，不超过动态上限
- [ ] 比率型资源（社稷值/灰烬志/朝贡序）范围锁定在0-100

### 效率折扣验证
- [ ] 社稷值=70时，资源产出显示为正常值的70%
- [ ] 社稷值=50时，显示为50%，以此类推

### 资源不足验证
- [ ] 资源不足时，相关行动按钮灰色不可用
- [ ] 显示"资源不足"提示

### 下游系统集成验证
- [ ] 军事行动正确消耗战械
- [ ] 意识形态行动正确消耗/增加灰烬志
- [ ] 外交行动正确消耗/增加朝贡序
- [ ] 内部稳定度变化正确反映到派系满意度

### 存档与恢复验证
- [ ] 存档包含所有7种资源的当前值
- [ ] 读档后资源数值正确恢复
