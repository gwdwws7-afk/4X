# A2 回合阶段引擎 (Turn Phase Engine)

> **状态**: Draft
> **作者**: GDC Framework / 游戏设计师
> **最后更新**: 2026-03-31
> **实现支柱**: P0（回合循环执行器）

## Overview 概述

回合阶段引擎是A1回合主循环的具体执行器，负责管理每个回合内6个阶段的切换、阶段内行动点的分配与消耗、以及阶段转换的逻辑。

本系统是纯执行系统——它不定义游戏规则，只负责"按照A1定义的顺序执行阶段"。

## Player Fantasy 玩家幻想

玩家在每个阶段感受到"这是行动的时候了"。

阶段切换时，玩家看到清晰的面板变化：阶段名称高亮，行动点重置，可选行动列表刷新。当行动点耗尽时，玩家知道"这阶段结束了，等下回合"。

## Detailed Design 详细设计

### Core Rules 核心规则

#### 阶段执行流程

```
回合开始
    ↓
阶段1：内政
    ├── 重置行动点（3点内政专用 + 2点通用）
    ├── 显示内政行动列表
    ├── 处理玩家行动
    ├── 阶段结束时：触发E系统维护检查
    ↓
阶段2：情报
    ├── 重置行动点（1点情报专用 + 2点通用）
    ├── 显示情报行动列表
    ├── 处理玩家行动
    ├── 阶段结束时：更新F1迷雾状态
    ↓
阶段3：经济
    ├── 重置行动点（2点经济专用 + 2点通用）
    ├── 显示经济行动列表（B1/B2/B3）
    ├── 处理玩家行动
    ├── 阶段结束时：B1/B2/B3更新
    ↓
阶段4：外交
    ├── 重置行动点（1点外交专用 + 2点通用）
    ├── 显示外交行动列表（C1/C2）
    ├── 处理玩家行动
    ├── 阶段结束时：C1/C2更新
    ↓
阶段5：军事
    ├── 重置行动点（3点军事专用 + 2点通用）
    ├── 显示军事行动列表（D1）
    ├── 处理玩家行动
    ├── 阶段结束时：D1争夺点积累
    ↓
阶段6：敌对行动
    ├── 玩家行动点耗尽（无行动点分配给AI）
    ├── 执行G势力AI框架决策
    ├── D5战争结算
    ├── B5经济结算
    ├── J胜负检测
    ├── I1事件生成
    ↓
回合结束
```

#### 行动点重置规则

```
每个阶段开始时：
- 专用行动点：重置为阶段基础值
- 通用行动点：若有剩余，不重置（可跨阶段使用）
  - 2点通用点可在任意阶段使用
  - 每回合结束时，剩余通用点清零

示例：
阶段1结束时：内政行动点已用完，通用点还剩1点
阶段2开始：情报专用点3点重置，但通用点只有1点可用（2-已用1）
```

#### 阶段切换条件

```
阶段切换触发：
1. 玩家主动结束阶段（行动点耗尽或手动结束）
2. 玩家点击"结束阶段"按钮
3. 回合时间到（若有时限设置）

自动切换：
- 若玩家5分钟内无操作，可设置自动结束当前阶段
- 玩家可在设置中关闭自动切换
```

### 阶段行动点分配

| 阶段 | 专用行动点 | 通用行动点 | 可用行动 |
|------|-----------|-----------|---------|
| 内政 | 3 | 2 | E内部政治系统相关 |
| 情报 | 1 | 2 | F1情报系统相关 |
| 经济 | 2 | 2 | B1/B2/B3相关 |
| 外交 | 1 | 2 | C1/C2相关 |
| 军事 | 3 | 2 | D1相关 |
| 敌对 | 0 | 0 | G AI行动、D5/B5/J/I1结算 |

### 特殊阶段机制

#### 紧急阶段跳过

```
若某阶段无任何可用行动（资源不足/无目标）：
- UI显示"本阶段无可用行动"
- 自动跳过至下一阶段
- 不消耗行动点

示例：
- 战械为0时，军事阶段大部分行动不可用
- 但玩家仍可选择"结束阶段"手动跳转
```

#### 阶段锁定

```
阶段锁定条件：
- 玩家已完成该阶段：锁定（可回顾不可修改）
- 当前回合已过该阶段：锁定
- 下一阶段未解锁：不显示

解锁条件：
- 前一阶段完成 → 解锁下一阶段
- 玩家可主动跳转（若行动点允许）
```

## Formulas 公式

### 核心计算

#### 公式1：可用行动点计算
```
可用行动点 = 阶段专用点 + min(剩余通用点, 2)

阶段专用点：固定值（3/1/2/1/3）
剩余通用点：本回合已使用通用点差值
```

#### 公式2：阶段时间限制
```
每阶段建议时间：5分钟
超时自动结束：可设置开关
自动结束前提示：30秒警告
```

## Data Interfaces 数据接口

### A2向其他系统提供的接口

```typescript
interface PhaseEngine {
    // 获取当前阶段
    getCurrentPhase(): Phase
    
    // 获取当前阶段行动点
    getAvailableActionPoints(phase: Phase): ActionPoints
    
    // 消耗行动点
    spendActionPoints(phase: Phase, points: number): boolean
    
    // 结束当前阶段
    endCurrentPhase(): PhaseTransitionResult
}

interface PhaseTransitionResult {
    nextPhase: Phase
    wasForced: boolean
    remainingActionPoints: ActionPoints
}
```

### A2从其他系统接收的调用

```typescript
// 由K UI框架调用
onPhaseChanged(phase: Phase)

// 由游戏状态查询
onQueryCurrentState(): GameState
```

## Tuning Knobs 调参旋钮

| 旋钮 | 默认值 | 可调范围 | 调整后果 |
|------|--------|---------|---------|
| 每阶段超时 | 5分钟 | 3-10分钟 | 影响游戏节奏 |
| 自动跳过无可用行动 | 是 | 是/否 | 影响流畅度 |
| 通用点跨阶段使用 | 是（2点上限） | 0-3点 | 影响策略灵活性 |
| 超时自动结束 | 可设置 | 开/关 | 影响节奏控制 |
