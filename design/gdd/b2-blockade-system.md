# B2 封锁与反封锁系统 (Blockade System)

> **状态**: Designed
> **作者**: GDC Framework / 游戏设计师
> **最后更新**: 2026-03-31
> **实现支柱**: P0（不对称博弈核心场景）, P1（代价系统驱动）

## Overview 概述

封锁与反封锁系统是瓦希德帝国核心困境的机制化表达。黄金领通过多层次封锁手段试图扼杀瓦希德，而瓦希德则通过反制手段维持生存。

本系统定义：封锁如何升级？什么触发封锁？封锁如何解除？反制手段何时可用？封锁对各势力的差异化影响？

封锁不是单一状态，而是一个多维度、分级别的动态系统。黄金领可以选择性地升级某个维度的封锁，也可以选择性地豁免某些渠道——这给了外交谈判筹码。

## Player Fantasy 玩家幻想

玩家感受到"四面楚歌但总有一线生机"的张力。

封锁不是铁板一块。每一层封锁都有裂缝：金融封锁下东土商盟愿意开商盟券渠道；能源禁运下北境共主愿意以货易货；海上封锁时灰色市场仍在运转。玩家的任务就是在这些裂缝中找到生存空间，并在适当时机用军事或外交手段主动打破封锁。

同时，当玩家采取某些激进行动（如威胁关闭火海咽喉），封锁可能升级。玩家需要计算：这次行动值得触发多少封锁？对手会承受多大损失？自己能否扛住？

## Detailed Design 详细设计

### Core Rules 核心规则

#### 封锁类型定义

| 封锁类型 | 英文 | 控制势力 | 影响的B1参数 | 升级触发条件 |
|----------|------|---------|------------|-------------|
| **能源禁运** | Energy Embargo | 黄金领 | 封锁折扣=0.2 | 瓦希德军事行动威胁到黄金领核心利益 |
| **金流封锁** | Financial Blockade | 黄金领 | 星币可得系数=0.2 | 瓦希德绕过SWIFT结算被抓获 |
| **二级封锁** | Secondary Blockade | 黄金领 | 对瓦希德贸易的国家受限 | 瓦希德与第三国贸易扩大 |
| **军事禁运** | Military Embargo | 黄金领 | 先进武器获取渠道关闭 | 瓦希德获得非西方武器 |
| **海上封锁** | Naval Blockade | 黄金领 | 节点畅通系数=0.0（咽喉） | 火海咽喉被威胁关闭 |

#### 封锁等级状态机

```
[无封锁]
    ↓瓦希德采取激进行动
[单边封锁]（只激活1-2种封锁手段）
    ↓瓦希德继续挑衅 OR 外交谈判失败
[多边封锁]（3-4种封锁手段）
    ↓瓦希德采取极端行动 OR 全面战争爆发
[全面封锁]（全部5种封锁手段）
    ↓
[封锁崩溃]（瓦希德打破封锁 OR 谈判成功解除部分）
```

#### 各封锁手段详解

**1. 能源禁运**

```
激活条件：
- 瓦希德军事行动威胁到黄金领能源利益（如攻击亲黄金领国家）
- 外交谈判破裂后黄金领主动升级

解除条件：
- 外交协议签署（至少是"冻结冲突"级别）
- 瓦希德在相关区域让步
- 黄金领国内政治变化（新政府愿意谈判）

影响范围：
- 瓦希德能源出口收入 × 封锁折扣(0.2)
- 但国内能源消耗不受影响
```

**2. 金流封锁**

```
激活条件：
- 瓦希德尝试使用SWIFT替代渠道（如加密结算、第三国转口）
- 被黄金领情报系统发现
- 外交谈判失败后黄金领主动升级

解除条件：
- 瓦希德承诺不使用替代结算
- 东土商盟/北境共主提供担保
- 黄金领单方面解除（罕见）

影响范围：
- 瓦希德星币可得系数 = 0.2
- 但商盟券/北境结算渠道不受影响
```

**3. 二级封锁**

```
激活条件：
- 瓦希德与某第三国贸易扩大（东土商盟、北境共主等）
- 黄金领对该第三国施压

解除条件：
- 第三国拒绝配合二级封锁（需要外交努力）
- 瓦希德减少与该第三国贸易

第三国应对二级封锁的反应矩阵：

| 第三国 | 经济依赖瓦希德 | 政治关系黄金领 | 最可能反应 |
|-------|--------------|--------------|-----------|
| 东土商盟 | 高（能源进口） | 一般 | **硬顶**（不怕施压） |
| 北境共主 | 中（武器市场） | 有限同盟 | **视情况**（评估风险） |
| 中立国 | 低 | 无 | **妥协**（不值得冒险） |

配合意愿计算：
配合意愿 = f(经济依赖度, 政治关系, 自身实力, 瓦希德给的补偿)
> 0.7 → 硬顶
0.3-0.7 → 视情况
< 0.3 → 妥协
```

**4. 军事禁运**

```
激活条件：
- 瓦希德获得非西方武器系统（如北境共主武器）
- 黄金领发现瓦希德军事现代化进展

解除条件：
- 瓦希德承诺不军事化（不可信）
- 军事技术谈判成功（罕见）
- 全面战争

影响范围：
- 只能通过北境共主渠道获取武器（价格高、种类有限）
- 无法获取西方先进武器
```

**5. 海上封锁**

```
激活条件：
- 瓦希德威胁关闭火海咽喉
- 黄金领认为瓦希德即将采取不可逆行动

解除条件：
- 瓦希德撤回威胁
- 双方达成海上通道协议
- 军事冲突后新平衡

影响范围：
- 火海咽喉节点畅通系数 = 0
- 瓦希德能源出口完全中断
- 全球油价暴涨（金帐合众趁机获利）
```

### 封锁升级机制

#### 封锁升级机制（混合模式）

```
封锁升级机制：
1. 瓦希德行动 → 积累"封锁压力值"（被动积累）
2. 压力值超过阈值(25/60/100) → 进入"封锁升级审议"
3. 黄金领AI评估：
   - 当前国内政治状况（强硬派 vs 温和派）
   - 升级封锁的代价（盟友配合度、全球油价影响）
   - 外交谈判可能性
4. AI选择：立即升级 / 警告后升级(2回合) / 暂不升级
```

| 瓦希德行动 | 积累封锁压力值 | 升级速度 |
|-----------|--------------|---------|
| 军事行动威胁能源通道 | +30 | 进入审议 |
| 使用SWIFT替代渠道 | +25 | 进入审议 |
| 与第三国贸易扩大 | +20 | 3-5回合后 |
| 获得非西方武器 | +15 | 进入审议 |
| 威胁关闭咽喉 | +40 | 立即进入审议 |

#### 封锁升级惩罚计算

```
封锁总压力 = Σ(各封锁类型压力值)

封锁压力表：
- 能源禁运: +30压力
- 金流封锁: +25压力
- 二级封锁: +20压力
- 军事禁运: +15压力
- 海上封锁: +40压力

封锁等级阈值：
- 无封锁: 0
- 单边封锁: 25
- 多边封锁: 60
- 全面封锁: 100
```

#### 封锁升级的蝴蝶效应

```
当封锁升级时：
1. 全球油价波动（海上封锁触发+30%油价）
2. 黄金领盟友响应（多边封锁时其他势力配合）
3. 瓦希德国内反应（社稷值下降）
4. 北境/东土态度变化（评估风险后决定是否继续合作）
```

### 反封锁机制

#### 反封锁手段矩阵

| 反制手段 | 应对封锁 | 触发条件 | 代价 |
|---------|---------|---------|------|
| **商盟券渠道** | 金流封锁 | 与东土商盟签贸易协议 | 汇率损失（70商盟券/火油 < 80金叶/火油） |
| **北境本币结算** | 金流封锁+军事禁运 | 与北境共主关系≥"有限同盟" | 政治依赖加深，价格不利的 |
| **灰色市场** | 军事禁运 | 每回合1-3次机会 | 成本+40%，走私曝光风险 |
| **能源换战械** | 军事禁运 | 北境共主愿意交易 | 北境银积累，换取有限武器 |
| **陆路出口** | 海上封锁 | 控制北境通道/两河联邦 | 效率-40% |
| **两河联邦中转** | 海上封锁 | 两河联邦与瓦希德友好 | 外交成本 |

#### 反封锁有效性公式

```
反封锁有效率 = 基础有效率 × 渠道稳定性系数 × (1 - 封锁强度折扣)

基础有效率：
- 商盟券渠道: 0.85（85%正常效率）
- 北境本币结算: 0.75（75%正常效率）
- 陆路出口: 0.60（60%正常效率）
- 灰色市场: 0.55（55%正常效率，但不稳定）

渠道稳定性系数：
- 首次使用: 1.0
- 持续使用5回合以上: 0.9（被逐渐发现模式）
- 被二级封锁针对: 0.5

封锁强度折扣：
- 单边封锁: -0.1
- 多边封锁: -0.3
- 全面封锁: -0.5
```

### 封锁与外交联动

#### 封锁作为外交筹码

```
黄金领可用封锁升级作为外交谈判筹码：
- "解除能源禁运，换取瓦希德从某区域撤军"
- "暂停金流封锁，换取瓦希德停止核计划"
- "不升级海上封锁，换取瓦希德开放能源过境"

瓦希德可用打破封锁作为外交筹码：
- "威胁关闭火海咽喉，换取西方解除能源禁运"
- "减少与北境合作，换取西方解除军事禁运"
- "开放能源过境，换取国际承认/投资"
```

#### 封锁解除的代价

| 解除封锁方式 | 所需代价 |
|------------|---------|
| 外交谈判解除（单边） | 瓦希德在相关区域让步 + 朝贡序提升 |
| 外交谈判解除（多边） | 重大让步（如放弃核计划）+ 朝贡序大幅提升 |
| 强制打破（军事） | 封锁崩溃，但触发全面战争风险 |
| **封锁崩溃（黄金领内部分歧）** | **事件触发 + 时间累积概率** |

#### 封锁崩溃机制（混合模式）

```
封锁崩溃触发机制：

基础概率（每回合）：
- 封锁第1-5回合: 0%
- 封锁第6-10回合: 2%/回合
- 封锁第11-20回合: 5%/回合
- 封锁20回合以上: 10%/回合

叠加事件触发（任一发生则立即判定）：
- 全球经济危机事件: +15%立即触发概率
- 黄金领政权更替: 30%概率直接崩溃
- 黄金领陷入其他战争: 25%概率封锁松动
- 瓦希德军事胜利（重大）: 20%概率黄金领重新评估
```

## Formulas 公式

### 核心计算

#### 公式1：封锁总压力
```
封锁总压力 = 能源禁运×30 + 金流封锁×25 + 二级封锁×20 + 军事禁运×15 + 海上封锁×40

封锁等级判定：
if 总压力 < 25: 无封锁
elif 总压力 < 60: 单边封锁
elif 总压力 < 100: 多边封锁
else: 全面封锁
```

#### 公式2：封锁对经济的影响
```
有效能源出口 = 基础产出 × 出口比例 × 封锁折扣(封锁等级) × 节点畅通系数
封锁折扣(无封锁) = 1.0
封锁折扣(单边封锁) = 0.6
封锁折扣(多边封锁) = 0.3
封锁折扣(全面封锁) = 0.1

星币可得系数 = 1.0 - (封锁压力 / 150)（下限0.05）
```

#### 公式3：反封锁有效率
```
反封锁有效率 = 基础有效率 × 渠道稳定性系数 × (1 - 封锁强度折扣)

渠道稳定性系数 = 
    首次使用: 1.0
    每持续使用3回合: -0.1（最低0.5）
    被二级封锁针对: ×0.5

封锁强度折扣 = 
    单边封锁: 0.1
    多边封锁: 0.3
    全面封锁: 0.5
```

#### 公式4：封锁升级风险评估
```
瓦希德行动封锁风险值 = Σ(该行动触发的封锁压力)
如果风险值 + 当前封锁压力 > 阈值(60) → 触发多边封锁警告

封锁风险缓释手段：
- 外交铺垫（提前与相关势力沟通）: -20%风险
- 替代方案准备（已有反封锁手段）: -30%风险
- 强硬行动（军事+外交同时施压）: +50%风险
```

## Edge Cases 边界情况

| 边界情况 | 处理方式 |
|---------|---------|
| 多边封锁下所有反制渠道都被针对 | 商盟券仍可用（因为东土不怕二级封锁），北境渠道降级但不完全关闭 |
| 封锁突然解除（如黄金领内乱） | 立即恢复正常，同时触发B1经济快速复苏计算 |
| 瓦希德同时威胁多个咽喉要道 | 封锁升级速度加快，每威胁1个额外咽喉 +20%升级风险 |
| 封锁触发全球油价暴涨 | 金帐合众趁机获利，抵消部分封锁效果（但瓦希德仍受损） |
| 封锁崩溃触发 | 全球经济危机/政权更替/其他战争/瓦希德军事胜利，可叠加时间概率加速 |
| 二级封锁针对东土商盟 | 东土商盟可以选择"硬顶"（减产但不中断贸易）或"妥协"（减少贸易量） |

## Data Interfaces 数据接口

### B2向其他系统提供的接口

```typescript
interface BlockadeSystem {
    // 获取当前封锁状态
    getBlockadeState(): BlockadeState
    
    // 获取特定封锁类型的状态
    getBlockadeTypeStatus(type: BlockadeType): boolean
    
    // 获取封锁总压力值
    getTotalBlockadePressure(): number
    
    // 获取封锁等级
    getBlockadeLevel(): BlockadeLevel
    
    // 计算某反制渠道的有效率
    getCountermeasureEfficiency(channel: CountermeasureChannel): number
    
    // 评估某行动的封锁升级风险
    assessBlockadeRisk(action: Action): RiskAssessment
}

interface BlockadeState {
    level: BlockadeLevel
    totalPressure: number
    activeBlockades: BlockadeType[]
    countermeasureStatus: CountermeasureStatus[]
    nextEscalationRisk: number
}
```

### B2从其他系统接收的调用

```typescript
// 由D1军事行动系统调用
onMilitaryActionTaken(action: MilitaryAction): BlockadeImpact

// 由C2外交协议系统调用
onDiplomaticPressureApplied(target: FactionId, pressure: number)
onAgreementSigned(agreement: Agreement): BlockadeRelief

// 由G势力AI框架调用（黄金领AI决策）
onAureanDominationDecisions(decisions: AureanDecision[])

// 由E内部政治系统调用
onInternalStabilityCrisis(level: CrisisLevel)  // 国内危机可能导致封锁暂时放松

// 由I1事件系统调用
onGlobalEvent(event: GlobalEvent)  // 全球事件可能影响封锁（如战争爆发）
```

## Tuning Knobs 调参旋钮

| 旋钮 | 默认值 | 可调范围 | 调整后果 |
|------|--------|---------|---------|
| 能源禁运封锁压力 | 30 | 20-40 | 影响封锁升级速度 |
| 金流封锁封锁压力 | 25 | 15-35 | 影响金融封锁有效性 |
| 二级封锁封锁压力 | 20 | 15-30 | 影响第三方合作意愿 |
| 军事禁运封锁压力 | 15 | 10-25 | 影响非西方武器获取难度 |
| 海上封锁封锁压力 | 40 | 30-50 | 影响咽喉博弈激烈程度 |
| 封锁升级阈值（单边→多边） | 25 | 20-35 | 影响封锁升级节奏 |
| 封锁升级阈值（多边→全面） | 60 | 50-75 | 影响全面封锁触发难度 |
| 反封锁基础有效率 | 0.75-0.85 | 0.6-0.95 | 影响反封锁手段吸引力 |
