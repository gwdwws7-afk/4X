# K UI框架 (User Interface Framework)

> **状态**: Designed
> **作者**: GDC Framework / 游戏设计师
> **最后更新**: 2026-03-31
> **实现支柱**: P3（信息呈现层）

## Overview 概述

UI框架是游戏的"门面"——所有系统产生的数据最终通过UI呈现给玩家。本系统定义UI的布局结构、面板设计、交互模式、以及信息显示规范。

本系统的核心设计目标是：让复杂信息简单化——玩家不需要阅读大量数字就能理解局势，所有关键信息应该一目了然。

## Player Fantasy 玩家幻想

玩家第一眼看到UI就知道"现在是什么情况"。

右上角的资源栏一目了然：火油多少、金叶多少、社稷值多少。地图上红色标注着危机区域，绿色标注着机会。回合结算时，一个清晰的账单告诉玩家"这回合净赚了还是净亏了"。所有信息都是"一看就懂"——不需要玩家去解读数字背后的含义。

### UI动画与反馈

```
行动确认反馈：
- 点击行动按钮：按钮缩小0.95→恢复正常 (100ms)
- 资源不足：按钮抖动 + 红色闪烁
- 行动成功：绿色勾号动画 + 音效
- 行动失败：红色叉号动画 + 音效

面板切换动画：
- 面板从右侧滑入 (200ms ease-out)
- 面板从左侧滑出 (150ms ease-in)
- 重叠面板：底层面板缩小至80%

数据变化动画：
- 资源增加：数字绿色闪烁 + 向上飘字 "+5"
- 资源减少：数字红色闪烁 + 向下飘字 "-3"
- 进度条变化：平滑过渡 (300ms)
```

## Formulas 公式

### 核心计算

#### 公式1：资源条颜色判定
```
颜色 = f(当前值 / 上限)

if 比例 > 0.5: 绿色
elif 比例 > 0.25: 黄色
else: 红色

特殊规则：
- 战械 < 3: 红色（即使比例>0.25）
- 社稷值 < 30: 红色
- 商盟券 即将过期(<3回合): 橙色闪烁
```

#### 公式2：趋势箭头判定
```
趋势 = f(当前回合 vs 上一回合)

if 增加值 > 0: ▲
elif 增加值 < 0: ▼
else: —

增加值计算 = 当前值 - 上回合值
```

## Edge Cases 边界情况

| 边界情况 | 处理方式 |
|---------|---------|
| 资源为0 | 显示"0"但进度条为空，数字显示红色 |
| 资源超过上限 | 进度条满格+溢出标注（如"72+20储）"） |
| 行动点为0 | 当前阶段所有行动按钮灰色不可用 |
| 多面板重叠 | ESC关闭最上层；连续按3次ESC关闭所有 |
| 结算报告过长 | 可滚动；默认折叠显示摘要 |
| 紧急事件弹出 | 立即显示在所有面板之上；需要处理后才能继续 |

## Data Interfaces 数据接口

### K UI的输入接口（读取各系统）

```typescript
interface UISystem {
    // 读取当前游戏状态
    getGameState(): GameStateForUI
    
    // 读取资源显示数据
    getResourceDisplay(): ResourceDisplay[]
    
    // 读取势力关系显示数据
    getRelationDisplay(): RelationDisplay[]
    
    // 读取节点详情
    getNodeDetails(nodeId: string): NodeDetailDisplay
    
    // 读取行动列表
    getActionList(phase: Phase): ActionDisplay[]
    
    // 读取结算报告
    getSettlementReport(): SettlementDisplay
    
    // 读取胜利进度
    getVictoryProgress(): VictoryProgressDisplay[]
}

interface GameStateForUI {
    turn: number
    gameTime: string
    currentPhase: Phase
    actionPointsRemaining: number
    actionPointsTotal: number
}
```

## Tuning Knobs 调参旋钮

| 旋钮 | 默认值 | 可调范围 | 调整后果 |
|------|--------|---------|---------|
| 资源条刷新频率 | 实时 | 按回合 | 影响性能/响应速度 |
