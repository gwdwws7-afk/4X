# I1 事件系统 (Narrative Events System)

> **状态**: Draft
> **作者**: GDC Framework / 游戏设计师
> **最后更新**: 2026-03-31
> **实现支柱**: P2（叙事驱动核心）

## Overview 概述

事件系统是游戏的叙事引擎，根据游戏状态生成叙事事件，为玩家提供选择和后果。

## Player Fantasy 玩家幻想

玩家感受到"每一个决定都在书写历史"。

当你看到"火海咽喉危机升级"的事件时，你知道这是一个关键时刻——你的选择将决定后续的发展。当你选择"增派兵力"时，你知道这可能让局势升级，但你别无选择。这就是历史，一个选择接着一个选择。

## Detailed Design 详细设计

### Core Rules 核心规则

#### 事件类型

| 事件类型 | 英文 | 触发频率 | 示例 |
|---------|------|---------|------|
| **叙事事件** | Narrative | 每3-5回合 | 黄金领内部分歧、区域危机升级 |
| **响应事件** | Response | 玩家行动后 | 军事胜利庆祝、经济困难抗议 |
| **随机事件** | Random | 每5-8回合 | 自然灾害、意外外交突破 |
| **终局事件** | Endgame | 胜负条件触发 | 胜利宣告、失败总结 |

#### 事件结构

```
事件 = {
    标题: string,
    叙事: string,
    选项: [
        {
            选项文字: string,
            消耗: ResourceCost,
            效果: EventEffect[],
            预期结果: string
        }
    ],
    默认选项: OptionId,
    超时回合: number
}
```

#### 事件触发条件

```
叙事事件触发：
- 回合数达到阈值
- 游戏状态满足条件

响应事件触发：
- 玩家行动完成后立即触发
- 军事行动 → 胜利/失败事件
- 外交行动 → 成功/失败事件

随机事件触发：
- 每回合5%概率
- 特定条件满足时概率提升
```

#### 叙事事件示例

```
示例1：咽喉危机升级
标题：「火海咽喉危机升级」
条件：回合10+，咽喉处于争夺状态
叙事：「黄金领宣布加强在波斯湾的军事存在，伊朗面临更大压力。」

选项：
[A] 保持克制，观察形势
    消耗：朝贡序-3
    效果：局势维持
[B] 增派兵力，强化防御
    消耗：战械-2，行动点-1
    效果：咽喉稳定度+10，封锁风险+20%
[C] 呼吁国际社会关注
    消耗：朝贡序-8
    效果：可能获得国际同情，概率40%
```

### 叙事与游戏联动

```
事件 → J胜负判定：
- 重大叙事事件可能影响胜利进度
- 如"抵抗叙事传播"事件：灰烬志+5

事件 → G AI框架：
- 事件可能触发AI重新评估
- AI根据事件调整策略

事件 → E内部政治：
- 叙事事件可能影响派系满意度
- 如"军事胜利"事件：激进派+5
```
