# D5 战争结算系统 (War Resolution System)

> **状态**: Designed
> **作者**: GDC Framework / 游戏设计师
> **最后更新**: 2026-03-31
> **实现支柱**: P0（节点控制核心判定）, P1（军事行动终点）

## Overview 概述

战争结算系统是每个回合结束时对节点争夺进行最终判定的系统。D1军事行动负责积累争夺点数，D5负责判定控制权归属。

本系统的核心职责是：在回合结算阶段，根据各方争夺点数、领先优势、战略价值，计算节点控制权的最终归属。D5不是D1的替代，而是D1的终点——每一次军事行动的效果，最终由D5结算确认。

## Player Fantasy 玩家幻想

玩家在每个回合的军事阶段布局争夺点数，看到自己的代理人行动、特种部队、不对称防御在争夺表上积累。回合结束时，D5告诉玩家："你这回合的努力是否换来了节点控制？"

好的结算结果让玩家感到"我的策略奏效了"——比如贝鲁特从中立变成了瓦希德控制，每回合多了2金叶收入。坏的结果让玩家思考"下回合我需要更多投入"——比如争夺点数够了但领先优势不够，只能维持深度争夺状态。

结算过程应该是透明可追溯的。玩家应该能清楚地看到：我的点数从哪来？对手的点数从哪来？为什么我没有获得控制权？

## Detailed Design 详细设计

### Core Rules 核心规则

#### 结算触发时机

```
回合结算顺序（A1回合主循环联动）：

回合结束
    ↓
1. D5 战争结算（最先执行——确定节点归属）
    ↓
2. B5 经济结算（基于D5确定的节点归属计算产出）
    ↓
3. C1 外交关系更新（基于D5结果）
    ↓
4. E 内部政治更新（基于D5结果）
    ↓
5. I1 叙事事件生成（基于D5结果）
```

#### 争夺点数计算

**各方争夺点数来源**：

```
争夺点数 = Σ(本回合所有行动对此节点的贡献)

贡献来源：
- 代理人行动：+20~40点（取决于战械投入）
- 特种部队打击：+40点
- 特种部队情报：无争夺点
- 不对称防御：+30点（防守成功）
- 咽喉威胁：+15点（存在即贡献）
- 敌方行动对我方节点：-XX点（抵消）

点数衰减：
- 每回合结束时，争夺点数 -20%（ attrition消耗）
- 若某势力连续3回合控制某节点，争夺点数稳定在100（不再衰减）
```

#### 控制权判定

**判定公式**：

```
若 瓦希德点数 > 100 且 瓦希德点数 - 对手点数 >= 30:
    → 瓦希德获得节点控制权
    → 节点状态变为"势力控制"
    → 触发归属变化事件

若 对手点数 > 100 且 对手点数 - 瓦希德点数 >= 30:
    → 对手获得节点控制权
    → 节点状态变为"势力控制"
    → 触发归属变化事件

若 双方点数差距 < 30:
    → 维持争夺状态
    → 进入下一回合争夺
    → 争夺点数不衰减（双方继续累积）

若 双方点数均 <= 100 且 差距 >= 30:
    → 维持当前状态（中立或原有控制）
    → 争夺点数按衰减规则处理
```

#### 特殊情况判定

**无主争夺（双方均未达到100点）**：
```
判定结果：
if 瓦希德点数 > 对手点数:
    → 瓦希德主导争夺（但非控制）
    → 争夺状态：初期或深度（取决于是否超过66点）
    → 产出效率：50%（初期）或 0%（深度）
else:
    → 对手主导争夺
    → 同上逻辑
```

**点数过载（双方均超过100点且差距<30）**：
```
判定结果：
→ 争夺白热化
→ 触发"拉锯战"事件
→ 双方在本回合均无法获得控制权
→ 争夺点数 +50（下一回合继续）
```

### 结算流程

```
D5 战争结算流程
│
├── 步骤1：收集所有争夺节点数据
│   ├── 读取H2节点系统所有争夺中节点
│   ├── 读取D1本回合所有军事行动记录
│   └── 按节点聚合各方争夺点数
│
├── 步骤2：计算各方最终点数
│   ├── 基础点数 = Σ(行动贡献)
│   ├── 衰减处理 = 基础点数 × 0.8（若有争夺历史）
│   └── 稳定加成 = 若连续3回合主导 +50
│
├── 步骤3：执行控制权判定
│   ├── 对每个争夺节点执行判定公式
│   ├── 判定结果：控制权转移 / 维持争夺 / 维持现状
│   └── 记录判定理由（透明可追溯）
│
├── 步骤4：计算结算效果
│   ├── 节点归属变化 → H2更新
│   ├── 产出效率变化 → B5计算收入
│   ├── 路线效率变化 → B3重新计算
│   └── 触发叙事事件 → I1
│
└── 步骤5：输出结算报告
    ├── 各节点判定结果明细
    ├── 资源变化汇总
    ├── 战略态势评估
    └── 警告与提示
```

### 节点类型对结算的影响

| 节点类型 | 控制权判定特殊规则 |
|---------|-----------------|
| **能源节点** | 争夺点数判定时有+10加权（战略价值高） |
| **咽喉要道** | 若咽喉被连续争夺3回合未决，触发"咽喉危机"事件（全局影响） |
| **贸易枢纽** | 控制权判定后立即计算金叶收入变化 |
| **意识形态中心** | 控制权转移影响灰烬志（抵抗叙事扩散范围变化） |

### 战争结算与外交联动

```
控制权转移触发外交反应：

瓦希德获得节点控制：
- 敌方（若为黄金领/圣火序）外交关系 -10
- 敌方（若为金帐合众）评估是否升级封锁
- 北境共主/东土商盟：重新评估与瓦希德的合作意愿
- 触发"抵抗轴心扩张"叙事事件

瓦希德失去节点控制：
- 社稷值 -5（国民不满）
- 朝贡序 -5（国际地位下降）
- 可能触发B2封锁升级（若丢失能源节点）
- 触发"战略挫折"叙事事件

维持争夺（未决）：
- 无外交关系变化
- 双方继续累积争夺点数
- 触发"僵局"叙事事件
```

## Formulas 公式

### 核心计算

#### 公式1：最终争夺点数
```
最终点数 = 基础点数 + 稳定加成 - 衰减

基础点数 = Σ(本回合所有行动对此节点的贡献)

稳定加成：
if 该势力连续3回合主导此节点争夺:
    稳定加成 = +50（上限100，防止无限滚雪球）

衰减：
if 有争夺历史（上一回合也有争夺）:
    衰减 = 基础点数 × 0.2
    最终点数 = 基础点数 - 衰减
else:
    衰减 = 0
    最终点数 = 基础点数

注：衰减按回合计算（A5游戏时钟以回合为基础单位）
```

#### 公式2：控制权判定
```
判定结果 = f(瓦希德点数, 对手点数, 当前控制方)

核心条件：
条件1：瓦希德点数 > 100 AND 瓦希德-对手 >= 30
条件2：对手点数 > 100 AND 对手-瓦希德 >= 30
条件3：双方差距 < 30（无论是否>100）
条件4：其他情况

执行顺序：
if 条件1: 瓦希德获得控制
elif 条件2: 对手获得控制
elif 条件3: 维持争夺，加剧拉锯
else: 维持现状（无方主导争夺）
```

#### 公式3：争夺优先级排序
```
当多个节点同时需要判定时，按以下优先级排序：

争夺优先级 = 战略价值 × (1 + 僵局加成) × (1 + 咽喉加成)

僵局加成：
- 连续2回合未决: +0.2
- 连续3回合未决: +0.5
- 连续4回合以上: +1.0

咽喉加成：
- 若节点为咽喉要道: ×1.5

按优先级从高到低执行判定
```

#### 公式4：咽喉危机触发
```
咽喉危机条件：
if 节点类型 == 咽喉要道:
    if 连续争夺回合数 >= 3 AND 无人获得控制权:
        触发咽喉危机事件

咽喉危机效果：
- 全球油价 +20%（立即）
- 封锁升级审议强制提前
- 各方争夺点数 +30（下一回合）
```

## Edge Cases 边界情况

| 边界情况 | 处理方式 |
|---------|---------|
| 争夺节点数量超过5个 | 按优先级排序，每回合最多结算5个；其余顺延至下一回合 |
| 节点在本回合内归属变化两次 | 以回合结束时的归属为准；中间的变化不追溯 |
| 判定时双方点数完全相等 | 维持原有状态；触发"僵局"叙事事件 |
| 争夺点数超过200 | 封顶为200；超出部分不计入 |
| 某方主动放弃争夺（撤回单位） | 视为点数清零；对手自动获得控制（若>100） |
| 全面战争触发 | D5暂停；全面战争结果直接跳转"灾难性后果"处理 |
| 第三方势力（灰烬众）争夺 | 灰烬众点数独立计算；主导区域瓦希德行动+10%；协同行动+15% |

## Data Interfaces 数据接口

### D5向其他系统提供的接口

```typescript
interface WarResolutionSystem {
    // 执行回合战争结算
    executeTurnResolution(): ResolutionResult
    
    // 获取所有争夺节点的判定结果
    getResolutionResults(): NodeResolution[]
    
    // 获取咽喉危机状态
    getChokepointCrisisStatus(): ChokepointCrisis[]
    
    // 获取结算报告
    getResolutionReport(): ResolutionReport
}

interface ResolutionResult {
    turnNumber: number
    resolutions: NodeResolution[]
    chokepointCrises: ChokepointCrisis[]
    diplomaticEffects: DiplomaticEffect[]
    resourceChanges: ResourceChange[]
    narrativeTriggers: NarrativeTrigger[]
}

interface NodeResolution {
    nodeId: string
    nodeName: string
    
    vashidPoints: number
    opponentPoints: number
    currentController: FactionId
    
    resolution: 'vashid_control' | 'opponent_control' | 'contested' | 'maintained'
    pointsToNext: number                    // 距下一门槛还差多少点
    
    isChokepointCrisis: boolean
    consecutiveContestTurns: number
    
    reasoning: string                       // 判定理由（透明可追溯）
}

interface ResolutionReport {
    summary: string
    controlChanges: ControlChange[]
    upcomingThreats: ThreatAssessment[]
    recommendations: string[]
}
```

## Tuning Knobs 调参旋钮

| 旋钮 | 默认值 | 可调范围 | 调整后果 |
|------|--------|---------|---------|
| 控制门槛点数 | 100 | 80-120 | 影响获得控制权难度 |
| 领先优势门槛 | 30 | 20-40 | 影响判定"决定性胜利"的标准 |
| 每回合争夺衰减 | 20% | 15%-25% | 影响长期争夺的点数消耗 |
| 稳定加成门槛 | 3回合 | 2-4回合 | 影响连续主导争夺的奖励 |
| 稳定加成点数 | +50 | +30~+70 | 影响稳定加成的强度 |
| 咽喉危机连续争夺回合 | 3回合 | 2-4回合 | 影响咽喉危机触发速度 |
| 咽喉危机争夺加成 | +30 | +20~+40 | 影响咽喉危机时点数加成 |
| 争夺优先级咽喉加权 | ×1.5 | ×1.2~×2.0 | 影响咽喉节点在判定中的优先级 |
| 每回合最大结算节点数 | 5 | 3-8 | 影响结算效率和延迟 |

**默认值已确定，待Alpha阶段平衡测试验证。**

## Acceptance Criteria 验收标准

### 基本判定验证
- [ ] 各方争夺点数正确汇总
- [ ] 判定公式正确执行（>100 + 领先30 = 控制）
- [ ] 衰减规则正确应用
- [ ] 稳定加成正确计算（连续3回合+50）

### 节点状态验证
- [ ] 控制权转移时H2正确更新
- [ ] 争夺状态正确转换（中立↔争夺↔控制）
- [ ] 产出效率在争夺状态下正确（50%或0%）

### 特殊判定验证
- [ ] 无主争夺（双方均未>100）时正确处理
- [ ] 拉锯战（双方>100且<30差距）正确触发
- [ ] 咽喉危机条件满足时正确触发

### 下游联动验证
- [ ] H2节点归属正确更新
- [ ] B5正确接收归属数据计算产出
- [ ] B3正确重新计算路线效率
- [ ] C1正确处理外交关系变化
- [ ] E正确处理社稷值变化
- [ ] I1正确触发叙事事件

### UI集成验证
- [ ] 结算报告正确显示各节点判定结果
- [ ] 判定理由透明可追溯
- [ ] 控制权转移时UI正确反馈
