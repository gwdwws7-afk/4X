# B5 经济结算系统 (Economic Settlement System)

> **状态**: Designed
> **作者**: GDC Framework / 游戏设计师
> **最后更新**: 2026-03-31
> **实现支柱**: P0（经济闭环验证）, P1（代价系统结算基础）

## Overview 概述

经济结算系统是回合结束时对所有经济活动进行汇总和净值计算的核心系统。它从B1（星币-能源）、B2（封锁）、B3（贸易网络）收集数据，结合A3（资源系统）的基础规则，产出本回合最终的资源变化快照。

本系统的核心职责是：确保所有资源变化符合游戏规则（不超上限、不为负），计算各系统之间的联动效果（如封锁影响贸易、贸易影响收入），并为下一回合提供正确的初始状态。

## Player Fantasy 玩家幻想

玩家在每个回合结束时，看到一个清晰的"经济账单"：

本回合能源出口净收入：金叶 +448
本回合军事采购：-200
本回合外交支出：-50
本回合封锁损失：-180（能源禁运导致）
本回合净变化：+18 金叶

玩家不应该感到困惑或惊讶——每一个数字都应该能追溯到具体的决策和事件。如果收入减少了，一定是封锁升级了或路线被切断了；如果支出增加了，一定是买了什么东西。透明的经济结算让玩家感到"我理解我的经济状况"。

## Detailed Design 详细设计

### Core Rules 核心规则

#### 结算流程总览

```
回合结算 - B5 经济结算
│
├── 阶段1：读取初始状态
│   ├── 读取A3资源系统当前值（回合开始时）
│   ├── 读取B1封锁状态（封锁等级、压力值）
│   ├── 读取B2全球油价
│   └── 读取B3贸易网络状态（各路线效率）
│
├── 阶段2：计算收入
│   ├── B1 能源出口收入
│   │   ├── 瓦希德高原产出（考虑封锁）
│   │   ├── 基尔库克产出（若控制）
│   │   └── 路线效率影响
│   ├── B3 贸易枢纽收入（金叶）
│   └── 其他收入（外交赠礼、事件等）
│
├── 阶段3：计算支出
│   ├── D1 军事行动支出（消耗战械、金叶）
│   ├── C2 外交行动支出（消耗朝贡序、金叶）
│   ├── C3 意识形态行动支出（消耗灰烬志、社稷值）
│   └── E 内部政治支出（消耗社稷值）
│
├── 阶段4：应用比率型资源效率折扣
│   ├── 社稷值折扣 → 影响所有资源产出
│   ├── 灰烬志加成 → 影响意识形态行动成功率
│   └── 朝贡序加成 → 影响外交谈判
│
├── 阶段5：边界检查与调整
│   ├── 累积型资源：不超过动态上限
│   ├── 比率型资源：锁定在0-100
│   ├── 消耗型资源：不为负
│   └── 战略储备池：按规则补充/消耗
│
└── 阶段6：产出结算报告
    ├── 资源变化明细
    ├── 经济健康度评估
    ├── 警告与提示
    └── 存档快照
```

#### 各系统结算职责

| 系统 | 负责计算 | 输出至B5 |
|------|---------|---------|
| **A3 资源系统** | 资源基础规则、上下限 | 资源当前值、动态上限 |
| **B1 星币-能源** | 能源收入、各渠道结算 | 能源出口量、各货币收入 |
| **B2 封锁系统** | 封锁等级、封锁折扣 | 封锁状态、封锁压力 |
| **B3 贸易网络** | 路线效率、贸易量分配 | 路线效率、可用贸易量 |
| **D1 军事行动** | 军事支出 | 战械消耗、金叶消耗 |
| **C2 外交协议** | 外交支出 | 朝贡序变化、金叶支出 |
| **E 内部政治** | 内部稳定消耗 | 社稷值变化 |

#### 结算优先级

当资源不足以支付所有支出时，按以下优先级结算：

```
优先级1：维持生存的必要支出
├── 军事行动（已承诺的，如正在进行的代理人行动）
└── 国内稳定（防止社稷值崩溃）

优先级2：维持经济运转的支出
├── 军事采购（补充战械库存）
└── 贸易渠道维护（商盟券协议等）

优先级3：发展性支出
├── 外交行动
├── 意识形态行动
└── 科技研发

优先级4：可选支出
└── 外交赠礼、面子工程等

低优先级支出在资源不足时被跳过，不触发失败但产生"未完成"状态
```

### 详细结算逻辑

#### 阶段2：收入计算

**能源出口收入计算**：

```
收入来源 = B1.能源出口收入(B3贸易量 × B1出口价格 × B2封锁折扣)

示例（本回合）：
B3可用贸易量 = 7.84单位（路线效率80%）
B1出口价格 = 80金叶/单位
B2封锁折扣 = 0.6（多边封锁）

能源出口收入 = 7.84 × 80 × 0.6 = 376.32 ≈ 376金叶
```

**贸易枢纽金叶收入计算**：

```
收入来源 = Σ(B3各贸易枢纽金叶产出)

各枢纽收入 = 基础产出 × (1 + 路线加成) × (稳定性/100) × 社稷值折扣

示例（本回合）：
贝鲁特：基础2金叶，路线加成0.2，稳定性80%，社稷值70%
贝鲁特收入 = 2 × 1.2 × 0.8 × 0.7 = 1.344 ≈ 1金叶
```

**总收入汇总**：

```
本回合总收入 = 
    能源出口收入（星币）
  + 贸易枢纽金叶收入
  + 外交赠礼（偶然事件）
  + 战械缴获（军事胜利）
  - 灰色市场成本（若有走私）
```

#### 阶段3：支出计算

**军事行动支出**：

```
支出 = Σ(D1各军事行动消耗)

示例（本回合）：
代理人行动（1点）：-2战械
特种部队（2点）：-3战械 -20金叶（情报费用）

本回合军事总支出：-5战械，-20金叶
```

**外交行动支出**：

```
支出 = Σ(C2各外交行动消耗)

本回合外交总支出：-30朝贡序，-50金叶
```

**意识形态行动支出**：

```
支出 = Σ(C3各意识形态行动消耗)

本回合意识形态总支出：-15灰烬志，-10社稷值
```

#### 阶段4：比率型资源效率折扣

**社稷值折扣**：

```
所有资源产出 = 基础产出 × (社稷值 / 100)

社稷值 = 70时：所有产出 × 0.7
社稷值 = 50时：所有产出 × 0.5
社稷值 = 30时：所有产出 × 0.3（危机状态）
```

**灰烬志加成**（影响C3意识形态行动）：

```
意识形态行动成功率 = 基础成功率 × (1 + 灰烬志/200)

基础成功率 = 30%
灰烬志 = 30时：30% × 1.15 = 34.5%
灰烬志 = 80时：30% × 1.4 = 42%
```

**朝贡序加成**（影响C2外交谈判）：

```
外交谈判加成 = 朝贡序 / 200

朝贡序 = 20时：+10%谈判成功率
朝贡序 = 80时：+40%谈判成功率
```

#### 阶段5：边界检查与调整

**累积型资源上限检查**：

```
if 火油 > 动态上限:
    超出部分 → 战略储备池（上限20）
    战略储备池已满 → 溢出消失
    显示警告："能源库存已满，建议增加出口或建造存储设施"

if 金叶 > 动态上限:
    超出部分 → 溢出消失
    显示警告："星币储备已达上限，建议增加采购"

if 商盟券 > 动态上限:
    超出部分 → 溢出消失
    显示警告："商盟券即将过期，建议使用"
```

**比率型资源边界检查**：

```
社稷值 = clamp(社稷值, 0, 100)
灰烬志 = clamp(灰烬志, 0, 100)
朝贡序 = clamp(朝贡序, 0, 100)

if 社稷值 < 20:
    触发国内危机事件
    封锁可能自动松动（黄金领评估是否继续封锁）
```

**消耗型资源检查**：

```
if 战械 < 0:
    该军事行动失败，回退消耗
    显示错误："战械不足，无法执行该行动"
    该行动已消耗的行动点不返还（决策代价）
```

### 经济健康度评估

每个回合结束时，B5输出一个经济健康度评分：

```
经济健康度 = f(储备充足度, 收入稳定性, 封锁压力, 增长趋势)

评分维度：
- 储备充足度：金叶储备能支撑几回合？战械库存够用吗？
- 收入稳定性：能源出口收入波动大吗？主要依赖哪个渠道？
- 封锁压力：当前封锁等级是什么？反封锁渠道有效性如何？
- 增长趋势：本回合vs上回合，收入/支出/储备趋势如何？

评分等级：
90-100：经济繁荣（绿色）
70-89：经济健康（浅绿）
50-69：经济一般（黄色）
30-49：经济压力（橙色）
0-29：经济危机（红色）
```

### 经济预警系统

```
预警等级1（提前3回合）：
触发条件：经济健康度 < 50 或 储备 < 2个月支出
提示：「经济承压，建议调整策略」

预警等级2（提前2回合）：
触发条件：经济健康度 < 35 或 储备 < 1个月支出
提示：「经济危机迫近，当前路线不可持续」

预警等级3（提前1回合）：
触发条件：经济健康度 < 20
提示：「下回合将出现负收入，请立即采取行动」

预警触发效果：
- UI显示警告图标（经济面板）
- 结算报告首页显示红色提示
- 建议行动列表（基于AI分析）
```

### 透支机制

```
透支允许：
- 允许金叶储备为负（短期周转困难）
- 透支上限：最多透支相当于1个月收入的金叶

透支惩罚：
- 透支利息：每回合透支余额 × 10% 额外消耗社稷值
- 透支期间：所有外交行动成功率 -15%

透支危机：
- 连续3回合透支：触发"债务危机"事件
  → 强制出售资产（如部分能源储备）
  → 或被迫接受不利外交条件
- 连续5回合透支：触发"经济崩溃"警告

透支清算：
- 若某回合末储备为正：立即补足
- 若连续5回合透支：触发"经济崩溃"警告
```

## Formulas 公式

### 核心计算

#### 公式1：本回合净资源变化
```
净资源变化 = Σ(收入) - Σ(支出) - Σ(损耗) - Σ(被动消耗)

被动消耗：
- 社稷值自然衰减：无（比率型资源无被动衰减）
- 商盟券过期：每10回合 × 50%
- 战略储备池上限：超出部分消失
```

#### 公式2：社稷值折扣系数
```
社稷值折扣 = clamp(社稷值 / 100, 0, 1)

所有资源产出 = 基础产出 × 社稷值折扣
```

#### 公式3：动态上限计算
```
累积型资源动态上限 = 基础上限 + Σ(上限加成来源)

加成来源：
- 外交协议：+5/份（每份贸易协议）
- 科技投资：+10/项（每项储罐技术）
- 建筑设施：+15/个（每座储油设施）

示例：
火油基础上限100 + 外交协议(+5) + 储罐科技(+10) + 2个储油设施(+30) = 145
```

#### 公式4：经济健康度评分
```
经济健康度 = 
    (储备充足度得分 × 0.25)
  + (收入稳定性得分 × 0.25)
  + (封锁压力得分 × 0.25)
  + (增长趋势得分 × 0.25)

储备充足度得分：
- 金叶储备 > 3个月支出: 100分
- 金叶储备 > 2个月支出: 70分
- 金叶储备 > 1个月支出: 40分
- 金叶储备 < 1个月支出: 10分

收入稳定性得分：
- 3个以上活跃渠道: 100分
- 2个活跃渠道: 70分
- 1个活跃渠道: 40分
- 0个活跃渠道: 10分

封锁压力得分：
- 无/单边封锁: 100分
- 多边封锁: 60分
- 全面封锁: 20分

增长趋势得分：
- 本回合收入 > 上回合: +30分
- 本回合收入 = 上回合: 0分
- 本回合收入 < 上回合: -30分
```

## Edge Cases 边界情况

| 边界情况 | 处理方式 |
|---------|---------|
| 收入为负（支出大于收入） | 金叶储备减少；若储备不足，显示"经济透支"警告 |
| 战略储备池已满 | 超出部分溢出消失；提示玩家"资源损失" |
| 社稷值降至0 | 触发国内崩溃事件；游戏进入特殊状态（可能直接失败） |
| 比率型资源超过100 | 锁定在100；超出部分不累积 |
| 消耗型资源为负 | 该行动失败，消耗的资源回退但不返行动点 |
| 结算过程中发生外交事件 | 事件效果计入下一回合结算（本回合已完成的不追溯） |
| 存档时正在结算 | 结算必须完成才能存档；存档包含完整结算后状态 |

## Data Interfaces 数据接口

### B5向其他系统提供的接口

```typescript
interface EconomicSettlementSystem {
    // 执行回合结算
    executeTurnSettlement(): SettlementResult
    
    // 获取当前经济健康度
    getEconomicHealth(): EconomicHealthScore
    
    // 获取资源变化明细
    getResourceChangeReport(): ResourceChangeReport
    
    // 获取经济预测（未来N回合趋势）
    getEconomicForecast(turns: number): EconomicForecast
}

interface SettlementResult {
    turnNumber: number
    timestamp: GameTimestamp
    resourceChanges: ResourceChange[]
    economicHealth: EconomicHealthScore
    warnings: EconomicWarning[]
    isComplete: boolean
}

interface ResourceChange {
    resourceType: ResourceType
    previousValue: number
    changeAmount: number
    newValue: number
    changeReason: string
}

interface EconomicHealthScore {
    overall: number              // 0-100
    grade: 'A' | 'B' | 'C' | 'D' | 'F'
    dimensions: {
        reserves: number
        stability: number
        blockadePressure: number
        growthTrend: number
    }
    trend: 'improving' | 'stable' | 'declining'
}
```

### B5从其他系统接收的调用

```typescript
// 由A1回合主循环调用（回合结束时）
onTurnEnd(turnNumber: number)

// 由各行动系统调用（本回合内的行动消耗）
onActionExecuted(action: GameAction, cost: ResourceCost)

// 由事件系统调用（偶然事件影响）
onEventOccurred(event: GameEvent, economicImpact: EconomicImpact)
```

## Tuning Knobs 调参旋钮

| 旋钮 | 默认值 | 可调范围 | 调整后果 |
|------|--------|---------|---------|
| 经济健康度权重-储备 | 25% | 15%-35% | 影响储备充足度重要性 |
| 经济健康度权重-稳定 | 25% | 15%-35% | 影响收入稳定性重要性 |
| 经济健康度权重-封锁 | 25% | 15%-35% | 影响封锁压力重要性 |
| 经济健康度权重-趋势 | 25% | 15%-35% | 影响增长趋势重要性 |

| 商盟券过期周期 | 10回合 | 8-12回合 | 影响囤积限制 |
| 透支利息 | 10%/回合 | 5%-15% | 影响透支惩罚力度 |
| 透支上限 | 1个月收入 | 0.5-2个月 | 影响短期周转空间 |
| 透支外交惩罚 | -15% | 10%-20% | 影响透支期间外交 |
| 经济预警周期 | 3/2/1回合 | 2-4回合 | 影响预警提前量 |

## Acceptance Criteria 验收标准

### 结算流程验证
- [ ] 回合结算按阶段1-6正确顺序执行
- [ ] 各系统数据正确读取（B1/B2/B3/A3）
- [ ] 结算结果与各系统本地计算一致

### 收入计算验证
- [ ] 能源出口收入 = B3贸易量 × B1价格 × B2封锁折扣
- [ ] 贸易枢纽金叶收入正确计算（基础×加成×稳定性×社稷值）
- [ ] 社稷值折扣正确应用

### 支出计算验证
