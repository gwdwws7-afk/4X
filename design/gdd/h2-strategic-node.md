# H2 战略节点系统 (Strategic Node System)

> **状态**: Designed
> **作者**: GDC Framework / 游戏设计师
> **最后更新**: 2026-03-31
> **实现支柱**: P0/P1/P2/P3（节点是所有势力博弈的核心载体）

## Overview 概述

战略节点系统是游戏世界的底层数据框架，定义所有可争夺的地理位置。与传统4X的六角格地图不同，本游戏的节点是地缘政治杠杆点——能源产地、咽喉要道、贸易枢纽、意识形态中心。控制关键节点即可影响势力消长。

本系统维护节点的位置数据、类型归属、资源产出、连接关系和控制状态。是H1战略地图的渲染数据源，也是D1军事行动、B3能源贸易、C1外交关系的核心数据提供者。

## Player Fantasy 玩家幻想

节点不是静态的"领土格子"，而是"可以失去也可以夺回来的战略资产"。

玩家在查看节点时，会意识到每一个节点背后都有三条线：这条线给我带来什么资源？这条线被切断会有什么后果？谁会因此受损，谁会因此得利？节点数据让玩家在发动军事行动前看到的不只是"我打下这个城"，而是"我切断了自己的能源命脉"或"我卡住了对手的贸易咽喉"。

## Detailed Design 详细设计

### Core Rules 核心规则

#### 节点数据模型

每个节点是一个包含以下属性的数据对象：

```
Node {
    id: string                    // 唯一标识符，如 "node_hormuz"
    name: string                  // 显示名称，如 "火海咽喉"
    type: NodeType                // 节点类型枚举
    region: Region                // 所属区域
    position: {x: float, y: float} // 在H1地图上的像素坐标（相对于地图中心）
    owner: FactionId              // 当前控制势力
    contested: boolean            // 是否处于争夺状态
    contestedBy: FactionId[]      // 正在争夺的势力列表
    
    // 类型特定属性
    resource产出: Resource产出[]  // 产出的资源类型和数量
    strategicValue: int           // 战略价值评分（影响AI优先级）
    controlPoints: int            // 占领所需控制点数
    
    // 连接关系
    connections: Connection[]     // 与其他节点的连线
    connectionType: ConnectionType // 连线类型：贸易路线/军事部署/影响力投射
    
    // 状态
    visibility: FogState          // 迷雾可见度
    stability: float              // 稳定度（0-100），影响产出效率和争夺难度
}
```

### 节点状态与转换

#### 控制权状态机

```
[中立] ←→ [争夺中] ←→ [势力控制]
  ↑           ↓           ↓
  ←←←←←←←←←←←←←←←←←←←←←←
  
中立：初始状态，无势力控制
争夺中：军事行动或外交压力导致控制权不确定
势力控制：某势力稳定控制，产出正常
```

#### 状态转换触发事件

| 当前状态 | 触发事件 | 下一状态 | 转换条件 |
|----------|---------|---------|---------|
| 中立 | D1军事行动开始 | 争夺中 | 军事单位进入节点 |
| 中立 | C2外交协议签署 | 势力控制 | 协议规定归属 |
| 争夺中 | D5战争结算完成 | 势力控制 | 某势力获得>50%控制点 |
| 争夺中 | 军事单位撤离 | 中立 | 30天内无军事存在 |
| 势力控制 | 敌方军事进攻 | 争夺中 | 敌方单位进入节点 |
| 势力控制 | 内部革命（E系统） | 争夺中 | 稳定性<20 |

#### 争夺机制

节点的争夺通过**控制点数**（Control Points）来衡量：

| 争夺阶段 | 控制点要求 | 说明 |
|----------|-----------|------|
| 初期争夺 | 1/3 CP | 军事存在，产出减半 |
| 深度争夺 | 2/3 CP | 产出中断，稳定性急速下降 |
| 占领待定 | 100% CP | 等待本回合结算判定 |

### Properties by Node Type 各类型节点属性

#### 能源节点

- **资源产出**：产出火油，数值因节点而异
- **封锁敏感性**：高——出口型能源节点受B2封锁系统影响大
- **争夺惩罚**：极高——失去能源节点直接减少火油收入
- **稳定性衰减**：当节点被封锁时，每回合稳定性-5

#### 咽喉要道

- **资源产出**：无直接资源产出
- **控制程度**：分三阶段——过路费（可选择性拦截）→ 封锁（切断特定航线）→ 关闭（完全切断，核选项级别）
- **外交代价**：控制程度越高，触发外交反应越强烈（联盟防御条款/D4核威慑联动）
- **争夺惩罚**：极高——是战争的最常见触发点

#### 贸易枢纽

- **资源产出**：产出金叶，受节点稳定性和连接线路影响
- **航线加成**：当连接的关键节点被控制时，金叶产出+20%
- **争夺惩罚**：中等——影响经济收入但不直接威胁生存
- **灰色市场**：贸易枢纽可作为B2封锁的绕过渠道（效率低于正常路线）

#### 意识形态中心

- **资源产出**：产出灰烬志，数值 = 基础产出 × (稳定性 / 100)，稳定性越高叙事越有说服力
- **叙事扩散**：控制意识形态中心可以提升相邻区域的抵抗叙事感染率
- **争夺惩罚**：中等——影响软实力积累速度
- **代理人联动**：D3代理人民事系统依赖意识形态中心的叙事扩散效果

## Formulas 公式

### 核心计算

#### 公式1：节点战略价值计算
```
有效战略价值 = 基础战略价值 × 稳定性系数 × 争夺惩罚系数
稳定性系数 = (稳定性 / 100)  // 稳定性越低，价值越低
争夺惩罚系数 = 
    无争夺: 1.0
    初期争夺: 0.7
    深度争夺: 0.3
```

#### 公式2：能源节点封锁影响
```
有效产出 = 基础产出 × 封锁系数 × 通道系数
封锁系数 = 
    无封锁: 1.0
    部分封锁: 0.5
    完全封锁: 0.0
通道系数 = 
    通道畅通: 1.0
    通道受压: 0.7
    通道中断: 0.0
```

#### 公式3：贸易枢纽航线加成
```
最终金叶产出 = 基础产出 × (1 + 路线加成)
路线加成 = Σ(每条活跃连接路线 × 0.2)
活跃连接 = 双方节点均无迷雾且无深度争夺
```

#### 公式4：意识形态中心叙事扩散
```
扩散半径 = 2 // 以节点为中心的区域数
叙事感染率 = 基础感染率 × (1 + 中心数量加成) × (稳定性 / 100)
中心数量加成 = 意识形态中心控制数 × 0.1
```

## Edge Cases 边界情况

| 边界情况 | 处理方式 |
|---------|---------|
| 节点同时被多势力争夺 | 多方同时积累控制点数，本回合结算时判定 |
| 咽喉要道被切断但有替代路线 | 替代路线效率-40%，咽喉价值部分转移 |
| 意识形态中心被封锁但产出灰烬志 | 通过C3替代渠道（远程叙事输出）效率-50% |
| 节点数据损坏或缺失 | 默认为"未知节点"，显示为灰色特殊节点 |
| 节点归属在回合中途变化 | 立即更新H1显示，产出按比例截断 |

## Data Interfaces 数据接口

### H2向H1提供的接口

```typescript
interface NodeSystem {
    // 获取所有节点数据（供H1渲染）
    getAllNodes(): Node[]
    
    // 获取某势力可见的节点列表
    getVisibleNodes(factionId: FactionId): Node[]
    
    // 获取某区域的所有节点
    getNodesByRegion(region: Region): Node[]
    
    // 获取节点间的连接关系
    getConnections(): Connection[]
    
    // 获取某节点的控制状态
    getControlState(nodeId: string): ControlState
}
```

### H2从其他系统接收的调用

```typescript
// 由D1军事行动调用
onMilitaryUnitMoved(unit: Unit, fromNode: Node, toNode: Node)
onControlPointsChanged(nodeId: string, factionId: FactionId, delta: number)

// 由F1情报系统调用
onVisibilityChanged(nodeId: string, newVisibility: FogState)

// 由E内部政治系统调用
onNodeStabilityChanged(nodeId: string, oldStability: float, newStability: float)

// 由C2外交协议系统调用
onNodeOwnershipChanged(nodeId: string, newOwner: FactionId)
```
