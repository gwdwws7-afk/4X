# C2 外交协议系统 (Diplomatic Protocol System)

> **状态**: Designed
> **作者**: GDC Framework / 游戏设计师
> **最后更新**: 2026-03-31
> **实现支柱**: P1（外交维度核心载体）, P2（叙事驱动）

## Overview 概述

外交协议系统是C1外交关系系统的下游延伸，负责具体协议的定义、签订、维护、违反和终止。每份协议都是一份有约束力的契约——签订后带来收益，但违反协议会有代价。

本系统的核心设计目标是：让协议成为"值得经营的资产"。签订协议需要投入，违反协议代价很大，长期维护协议需要持续投入。但协议也带来实实在在的好处——更好的贸易条件、军事合作、外交保护。

## Player Fantasy 玩家幻想

玩家在外交谈判桌上感受到"每一份协议都是一场赌博"。

你花了大价钱与北境共主签订了军事技术合作协议，这意味着每回合能获得更先进的战械。但这份协议让你对北境的依赖加深——当他们涨价时你更难拒绝。协议是活的——需要维护，需要履行承诺，需要在对手违约时决定是忍让还是报复。

好的协议策略是"组合拳"：与东土商盟签订贸易协议保证经济命脉，与北境共主签订军事协议获得武器来源，与灰烬众签订协调协议巩固同盟。但每一份协议都是潜在的弱点——过度依赖某一方会让你在关键时刻受制于人。

## Detailed Design 详细设计

### Core Rules 核心规则

#### 协议类型定义

| 协议类型 | 英文 | 可签订对象 | 基础关系要求 | 行动点成本 | 朝贡序成本 |
|---------|------|-----------|------------|-----------|-----------|
| **双边本币结算** | Currency Settlement | 东土商盟、北境共主 | 一般(+40)以上 | 1 | 10 |
| **军事技术合作** | Military Cooperation | 北境共主 | 友好(+41)以上 | 1 | 15 |
| **抵抗轴心协调** | Axis Coordination | 灰烬众 | 友好(+41)以上 | 1 | 8 |
| **能源过境协议** | Energy Transit | 金帐合众、北境共主 | 一般(+40)以上 | 1 | 10 |
| **中立保障** | Neutrality Guarantee | 中立国 | 冷淡(-40)以下 | 2 | 20 |
| **联合防御条款** | Defense Pact | 灰烬众 | 同盟(+81) | 2 | 25 |

#### 协议效果矩阵

| 协议类型 | 直接效果 | 持续效果 | 触发条件 |
|---------|---------|---------|---------|
| **双边本币结算** | 商盟券/北境银渠道解锁 | 每回合贸易获得优惠 | 永久（直到违反） |
| **军事技术合作** | 每回合+1~2战械（取决于协议等级） | 武器购买价格9折 | 永久（直到违反） |
| **抵抗轴心协调** | 灰烬众协同行动效率+20% | 联合军事行动解锁 | 永久（直到违反） |
| **能源过境协议** | 指定节点可作为替代贸易路线 | 路线效率+15% | 永久（直到违反） |
| **中立保障** | 遭受攻击时大国保证干预 | 敌方攻击代价+20% | 协议有效期内 |
| **联合防御条款** | 灰烬众在遭受攻击时自动援助 | 获得2~3战械/回合援助 | 永久（直到违反） |

#### 协议等级

部分协议类型有不同等级，代表合作深度：

```
军事技术合作等级：
- L1基础：每回合+1战械，武器价格95折
- L2进阶：每回合+2战械，武器价格9折，可购买L2装备
- L3高级：每回合+3战械，武器价格85折，可购买L3装备

协议等级解锁条件：
- L2：与北境共主关系≥友好(+60)，且协议维持2回合以上
- L3：与北境共主关系≥同盟(+80)，且协议维持5回合以上
```

#### 协议生命周期

```
[协议签订] → [协议生效] → [协议维护] → [协议到期/违反/终止]
                              ↓
                        每回合检查：
                        - 是否满足维持条件
                        - 是否有违反行为
                        - 是否触发特殊事件
```

**协议签订流程**：
```
1. 玩家选择协议类型和目标对象
2. C2检查：关系是否满足要求
3. C2检查：对方是否有意愿（AI评估）
4. 计算签订成本（关系修正后的朝贡序消耗）
5. 消耗资源，签订协议
6. C1关系提升（+10~+15）
7. 协议生效
```

**协议维持条件**：
```
每回合维持检查：
- 双方关系是否仍在门槛以上（若低于门槛，协议进入"不稳定"状态）
- 是否履行协议承诺（如未履行，触发违反判定）
- 是否有外部事件影响协议效力

不稳定状态（协议降级/终止前兆）：
- 协议效果降低50%
- UI显示警告
- 玩家有2回合窗口修复关系
```

**协议违反判定**：
```
违反触发条件：
- 主动违反：玩家采取与协议精神相悖的行动
  - 签订中立保障后攻击某国
  - 签订抵抗轴心协调后不支持灰烬众
- 被动违反：未能履行协议义务
  - 能源过境协议后切断能源输送
  - 军事技术合作后未能支付"费用"

违反后果：
- C1关系大幅下降：-20~-40
- 协议立即终止
- 触发"背信弃义"叙事事件（I1）
- 对方可能采取反制措施
- 未来与该势力重签协议成本×2
```

**协议到期机制**：
```
有期限协议（自动续签）：
- 中立保障：10回合后到期
- 能源过境协议：每5回合重新谈判
- 到期前2回合：系统自动评估是否续签
  → 评估标准：关系是否仍满足门槛、对方AI意愿、续签成本变化
  → 结果：自动续签 / 提前通知玩家重新谈判 / 自动终止

无期限协议（永久）：
双边本币结算、军事技术合作、抵抗轴心协调、联合防御条款
- 无需续签
- 每回合维护检查仍然进行
- 违反或主动终止才结束
```

**谈判机制**：
```
外交行动"协议签订" = 谈判过程：
- 玩家选择协议 → 消耗行动点和资源 → 即时签约/被拒绝
- 若AI拒绝：关系-5，行动点不返还
- 拒绝后需等待2回合才能重试（防止反复刷谈判）
```

### 协议与游戏机制联动

#### 协议-B1联动

```
双边本币结算协议：
- 签订后：B1解锁对应结算渠道
- 维持期间：兑换汇率享受优惠（商盟券+5%，北境银+10%）
- 违反/终止：渠道立即关闭（已在处理中的交易完成）

能源过境协议：
- 签订后：B3解锁对应替代路线
- 维持期间：路线效率+15%
- 违反/终止：路线立即中断
```

#### 协议-D1联动

```
军事技术合作：
- 签订后：D1可用"北境渠道"购买战械
- 维持期间：购买价格折扣
- 违反/终止：渠道关闭

抵抗轴心协调：
- 签订后：D1与灰烬众协同行动效率+20%
- 协同行动消耗降低
- 违反/终止：协同效果消失

联合防御条款：
- 签订后：D1若遭受攻击，灰烬众自动响应
- 响应效果：每回合提供2~3战械援助
- 违反/终止：防御失效
```

#### 协议-C1联动

```
协议签订时：
- C1立即关系提升：+10~+15
- 触发冷却期机制（3回合）

协议维持期间：
- 长期合作（每5回合）：关系额外+2
- 关系降至"冷淡"以下：协议进入不稳定状态

协议违反时：
- C1立即关系下降：-20~-40
- 触发冷却期机制（3回合）
- 重签成本×2
```

### 特殊协议机制

#### 中立保障

```
中立保障是特殊的"保护协议"：

签订条件：
- 目标：任意大国愿意担保
- 成本：需要在该大国身上大量外交投资
- 时机：对方需要"展示"保护能力

效果：
- 遭受攻击时，担保方有50%概率干预
- 干预效果：攻击方遭受-20%效率惩罚
- 担保方：北境共主（愿意担保但价格高）

局限：
- 不防止主动攻击（你先打别人，别人不管）
- 不保证100%干预（只是概率）
- 担保方可能"要价"越来越高
```

#### 协议组合与叠加规则

```
同类协议（同一类型）：
- 不叠加，仅保留最高等级
- L2和L1同时签 = 仅L2生效
- UI提示："已存在同类协议（L1），升级为L2？"

不同类型协议（即使同一对象）：
- 效果叠加
- 风险叠加（违约代价更大）

示例组合：
"经济生命线"：
- 双边本币结算（东土）+ 能源过境协议
- 效果：能源出口渠道完全绕过星币体系

"军事同盟"：
- 军事技术合作（北境）+ 抵抗轴心协调（灰烬众）
- 效果：战械供应稳定，代理人网络协同

"孤立风险"（负面）：
- 同时签订多个北境协议
- 效果：过度依赖北境，政治灵活性下降
```

#### 谈判机制

```
外交行动"协议签订" = 谈判过程：

- 若AI拒绝：关系-5，行动点不返还
```

## Formulas 公式

### 核心计算

#### 公式1：协议签订成本
```
签订成本 = 基础朝贡序消耗 × 关系修正 × 势力修正

关系修正：
- 同盟(>+80)：×0.6
- 友好(+41~+80)：×0.8
- 一般(-40~+40)：×1.0
- 冷淡(-80~-41)：协议不可用
- 敌对(<-80)：协议不可用

势力修正（目标相关）：
- 黄金领：×1.5（最难签）
- 圣火序：×1.3
- 金帐合众：×1.1
- 北境共主：×1.0
- 东土商盟：×0.9
- 中立国：×1.2
```

#### 公式2：协议维持检查
```
每回合维持检查：

关系门槛检查：
if 当前关系 < 最低关系要求:
    协议进入不稳定状态
    效果减半
    触发2回合修复窗口

若2回合内关系未恢复：
    协议终止
    触发违反后果

违约检查：
if 主动采取违反协议的行动:
    立即触发违反后果
    终止协议
```

#### 公式3：协议效果计算
```
协议效果 = 基础效果 × 等级系数 × 稳定性系数

等级系数：
- L1：×1.0
- L2：×1.5
- L3：×2.0

稳定性系数：
- 正常状态：×1.0
- 不稳定状态：×0.5
```

#### 公式4：重签协议成本
```
重签协议成本 = 原始签订成本 × 重签系数

重签系数：
- 主动终止：×1.5
- 被动违反后重签：×2.0
- 被动违反（对方先违约）：×0.8（合理补偿）
```

## Edge Cases 边界情况

| 边界情况 | 处理方式 |
|---------|---------|
| 协议签订后关系骤降 | 进入不稳定状态，2回合修复窗口 |
| 同时签订多个北境协议 | 允许，但触发"过度依赖"警告 |
| AI主动提出协议谈判 | G框架控制AI主动提出协议的频率 |
| 协议到期时关系刚达标 | 续签可正常进行 |
| 协议一方政权更替 | 协议自动进入不稳定状态（新政权可能不认） |
| 协议与协议效果冲突 | 后签的协议优先；或显示冲突警告 |

## Data Interfaces 数据接口

### C2向其他系统提供的接口

```typescript
interface DiplomaticProtocolSystem {
    // 获取所有有效协议
    getActiveProtocols(): Protocol[]
    
    // 获取某势力的协议
    getProtocolsByFaction(factionId: FactionId): Protocol[]
    
    // 签订协议
    signProtocol(
        protocolType: ProtocolType,
        targetFaction: FactionId,
        level?: number
    ): SignResult
    
    // 终止协议（主动）
    terminateProtocol(protocolId: string): TerminateResult
    
    // 获取协议效果
    getProtocolEffects(protocolId: string): ProtocolEffect[]
    
    // 检查维持状态
    checkProtocolStability(): StabilityReport[]
}

interface Protocol {
    protocolId: string
    type: ProtocolType
    level: number
    parties: FactionId[]           // 签署方
    signedTurn: number             // 签署回合
    expiresTurn?: number           // 到期回合（若有）
    stability: 'stable' | 'unstable' | 'critical'
    effects: ProtocolEffect[]
}
```

### C2从其他系统接收的调用

```typescript
// 由C1外交关系系统调用
onRelationChanged(factionId: FactionId, oldRelation: number, newRelation: number)

// 由G势力AI框架调用
```
