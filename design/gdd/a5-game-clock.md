# A5 游戏时钟 (Game Clock)

> **状态**: Designed
> **作者**: GDC Framework / 游戏设计师
> **最后更新**: 2026-03-31
> **实现支柱**: P2（帝国演化依赖时间推进）

## Overview 概述

游戏时钟是游戏的时间引擎，负责推进游戏时间、检测时间上限、向所有下游系统发送时间推进信号。它是回合主循环（A1）的下游系统，每回合结束后接收触发信号，将游戏时间推进0.5年。

游戏从2028.0年开始，共24回合（2040.0年），时间到达上限后强制终局。

## Player Fantasy 玩家幻想

玩家感受到时间的重量。

每一回合都是半年的决策窗口。2028年时，帝国看起来还牢不可破；到2035年，裂缝已经开始出现；到2040年，历史在玩家手中定格。时间是不可再生资源——错过了某个窗口，就永远错过了。

玩家在时间压力下做决策：现在不做，半年后就更难做了。每一回合结束时的紧迫感，和每一回合开始时的规划感，共同构成"时间在流逝"的情感核心。

**设计目标**：玩家在每回合开始时感到"我还有时间"；在游戏后期感到"时间不多了"。

## Detailed Design 详细设计

### Core Rules 核心规则

#### 时间基本参数

| 参数 | 值 |
|------|---|
| 起始时间 | 2028.0 |
| 每回合推进 | 0.5年 |
| 总回合数 | 24回合 |
| 结束时间 | 2040.0 |
| 显示格式 | "2028.0" / "2028.5" / "2029.0" |

#### 时间推进流程

```
回合主循环触发回合结算
    ↓
游戏时钟接收到推进信号
    ↓
当前时间 = 当前时间 + 0.5年
回合数 = 回合数 + 1
    ↓
检测时间上限：回合数 > 24 ？
    ↓
是 → 触发强制终局（J胜负判定系统）
    ↓
否 → 向所有下游系统发送"新回合开始"信号
```

#### 显示格式

- 格式："年份.回合奇偶性"
- 2028.0 = 2028年上半年（第1-2回合）
- 2028.5 = 2028年下半年（第3-4回合）
- 以此类推

#### 暂停机制

- 玩家可随时切换暂停/继续状态
- 暂停状态下，时钟停止，不发送时间推进信号
- 暂停时玩家仍可查看地图、资源面板、情报
- 暂停时玩家不可执行任何消耗行动点数的行动
- 回合结束后自动解除暂停状态

#### 自动存档触发

- 每回合结算时，游戏时钟自动向存档系统（A4）发送存档信号
- 存档具体逻辑由A4存档系统负责
- 游戏时钟只提供触发信号，不管理存档数据

### States and Transitions 状态与转换

| 状态 | 说明 | 可转换到 |
|------|------|---------|
| **运行中** | 时钟正常推进，每回合+0.5年 | 暂停 |
| **暂停** | 时钟停止，不推进，不发送信号 | 运行中 |
| **终局** | 第24回合结束后，时钟停止 | — |

| 系统 | 依赖性质 | 依赖内容 |

#### 数据接口

本系统向其他系统提供：
- `getGameTime()` — 获取当前游戏时间（格式：2028.0）
- `getTurnNumber()` — 获取当前回合数（1-24）
- `isPaused()` — 查询时钟是否暂停
- `pause()` / `resume()` — 暂停/继续时钟

## Formulas 公式

### 核心计算

#### 公式1：时间推进
```
当前时间 = 起始年份 + (回合数 × 0.5)
```

#### 公式2：回合奇偶性判断
```
if 回合数 % 2 == 1: 显示 ".0"（上半年）
if 回合数 % 2 == 0: 显示 ".5"（下半年）
```

#### 公式3：时间上限检测
```
if 回合数 > 24: 触发强制终局
```

## Edge Cases 边界情况

| 边界情况 | 处理方式 |
|---------|---------|
| 玩家在暂停状态下尝试执行行动 | 行动按钮灰色不可用，提示"请先解除暂停" |
| 存档损坏导致时间数据丢失 | 存档系统（A4）负责处理，时钟重置为存档中的时间 |
| 回合数异常（负数或非整数） | 时钟系统进行校验，不符合则强制重置为1 |
| 时间到达2040.0后玩家尚未做出决策 | 强制终局立即触发，玩家当前阶段被中断 |

## Tuning Knobs 调参旋钮

| 旋钮 | 默认值 | 可调范围 | 调整后果 |
|------|--------|---------|---------|
| 起始年份 | 2028.0 | 2020-2035 | 过高→游戏时间太短；过低→节奏太慢 |
| 每回合年数 | 0.5年 | 0.25-1年 | 0.25→节奏太慢（48回合）；1→节奏太快（12回合） |
| 总回合数 | 24 | 20-30 | 过高→游戏过长；过低→胜利条件难以达成 |
| 结束年份 | 2040.0 | 计算得出 | 由起始年份+总回合数×每回合年数决定 |

**默认值已确定。**
