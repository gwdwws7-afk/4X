# 回合主循环 (Turn-Based Main Loop)

> **状态**: 设计中
> **作者**: GDC Framework / 游戏设计师
> **最后更新**: 2026-03-31
> **实现支柱**: P0/P1/P2/P3（所有支柱都依赖此系统驱动）

## Overview 概述

回合主循环是整个游戏的驱动心脏。它定义了每回合的结构：玩家如何分配行动点数、如何与敌对AI交替行动、游戏时钟如何推进。所有其他系统（经济、外交、军事、派系）都在这个框架内运转。

每回合代表现实中的6个月（游戏时间2028-2040，共约24回合）。玩家在6个阶段中消耗11个行动点数（混合制）来执行决策，然后AI敌对势力响应行动，最后进入下一回合。

## Player Fantasy 玩家幻想

11点永远不够用。

这是玩家在这个系统中最核心的感受：每回合结束，玩家总会有"这回合我还能做得更多"的感觉。选择在军事上投入3点，就意味着外交少了一次谈判的机会；选择了激进的意识形态输出，就要承受内部温和派的不满。

这种"稀缺感"不是设计的缺陷，而是游戏体验的核心。抵抗轴心的本质是以弱胜弱——资源永远不足，决策永远需要取舍。玩家在"什么都想做但什么都做不完"中，被迫做出优先级判断，感受到不对称博弈的窒息感。

当玩家在某一回合成功用有限的点数同时解决了内部危机、突破了经济封锁、又在军事上取得了进展，那种"四两拨千斤"的成就感就是最强的正反馈。

**设计目标**：玩家在每回合结束时感到"我做了我能做的一切，但总有遗憾"——而不是"我轻松搞定了所有事"或"点数根本不够做一件有意义的事"。

## Detailed Design 详细设计

### Core Rules 核心规则

#### 点数分配（混合制）

| 阶段 | 基础配额 | 可用行动类型 |
|------|---------|------------|
| 内政 | 2点 | 派系平衡管理、资源分配 |
| 情报 | 1点 | 侦察、渗透、反情报、暗部行动 |
| 经济 | 2点 | 封锁应对、灰色市场、双边结算、制裁耐受投资 |
| 外交 | 1点 | 双边谈判、协议签订、文明交流 |
| 军事 | 3点 | 代理人行动、特种部队、咽喉威胁、不对称防御 |
| 通用 | 2点 | 任意阶段使用，不限于上述类型 |
| **合计** | **11点/回合** | |

#### 回合基本流程

```
① 内政阶段（玩家）
    ↓
② 情报阶段（玩家）
    ↓
③ 经济阶段（玩家）
    ↓
④ 外交阶段（玩家）
    ↓
⑤ 军事阶段（玩家）
    ↓
⑥ 敌对行动阶段（AI统一响应）
    ↓
回合结算 + 游戏时钟前进6个月
    ↓
下一回合
```

#### 阶段内行动规则

- 每个阶段内可执行**多次行动**，自由分配该阶段点数
- 行动实时生效，不存在"取消窗口"
- 每个行动显示明确的代价预览（社稷值代价/派系影响/国际反应）
- 进入下一阶段后，上一阶段剩余点数自动作废，不可跨阶段转移
- 通用点数可在任意阶段使用，不受阶段类型限制

#### AI敌对行动规则

- 玩家5个阶段执行期间，AI**不实时响应**（信息不对称）
- AI在第6阶段统一响应一次，基于当前全局状态做决策
- AI行动对玩家可见，但不显示推理过程
- 玩家对AI无信息优势，但对AI行动有可预测性

### States and Transitions 状态与转换

| 状态 | 说明 | 可转换到 |
|------|------|---------|
| **内政** | 玩家执行内政行动 | 情报 或 直接进入任意后续阶段 |
| **情报** | 玩家执行情报行动 | 经济 或 直接进入任意后续阶段 |
| **经济** | 玩家执行经济行动 | 外交 或 直接进入任意后续阶段 |
| **外交** | 玩家执行外交行动 | 军事 或 直接进入任意后续阶段 |
| **军事** | 玩家执行军事行动 | 敌对行动 |
| **敌对行动** | AI统一响应 | 回合结算 |
| **回合结算** | 资源回收、派系更新、事件触发 | 下一回合-内政 |
| **游戏结束** | 胜负判定触发 | — |

#### 跳阶段规则

玩家可以跳过中间阶段直接进入后续阶段（前提是有足够剩余点数），但不能跳回之前的阶段。

#### 游戏结束

游戏结束由J胜负判定系统触发，不属于回合主循环的状态转换范畴。

### Interactions with Other Systems 与其他系统的交互

#### 本系统向其他系统提供的数据接口

| 下游系统 | 本系统提供的数据 |
|---------|----------------|
| A2 回合阶段引擎 | 当前阶段(phase)、阶段序号(1-6) |
| A4 存档系统 | 每回合结束生成存档快照 |
| A5 游戏时钟 | 回合结束后推进6个月(2028→2028.5→2029...) |
| B1 星币-能源 | 每回合触发一次经济结算 |
| B2 封锁系统 | 封锁状态随回合更新 |
| C1 外交关系 | 外交关系值随回合衰减或增长 |
| D1 军事行动 | 玩家当前可用军事点数 |
| E 内部政治 | 每回合触发派系满意度更新 |
| F1 情报系统 | 每回合触发情报刷新 |
| G 势力AI | 每回合触发AI决策计算 |
| J 胜负判定 | 每回合触发胜利条件检测 |
| K UI框架 | 当前阶段、剩余点数、可用行动列表 |

#### 数据所有权原则

回合主循环拥有以下核心状态的唯一写入权：
- 当前处于哪个阶段
- 玩家剩余行动点数

所有下游系统只有读取权，无修改权。如有系统需要修改这些状态，必须通过回合主循环的明确接口，不允许跨系统直接修改。

## Formulas 公式

### 核心计算

#### 公式1：阶段可用点数
```
阶段剩余点数 = 阶段基础配额 + 阶段剩余通用点数
阶段剩余通用点数 = 通用点数配额 - 已使用通用点数
```

- 阶段基础配额：该阶段的基础行动点数（例：军事=3，内政=2）
- 通用点数配额：每回合固定2点
- 已使用通用点数：本回合内玩家已花费在任意阶段的通用点数总和

#### 公式2：游戏时间推进
```
当前时间 = 起始年份 + (回合数 × 0.5年)
回合数范围：1-24（2028年-2040年）
```

#### 公式3：回合年数显示
```
if 当前回合 % 2 == 1: 显示"H1 202X年上"
if 当前回合 % 2 == 0: 显示"H2 202X年下"
```

每回合推进0.5年（半年），每2回合=1个完整年份。

游戏时间从2028年开始，共24回合，约12个游戏年份。

## Edge Cases 边界情况

| 边界情况 | 处理方式 |
|---------|---------|
| **0点数时** | 玩家仍可进入下一阶段（跳阶段），但不能执行任何消耗点数的行动 |
| **点数不足** | 行动按钮灰色不可用，显示"点数不足"提示 |
| **玩家跳过全部阶段** | 允许，但AI仍在第6阶段响应，玩家放弃本回合所有行动机会 |
| **AI响应时玩家离线**（多人模式） | AI仍执行响应，回合继续推进 |
| **游戏存档损坏** | 存档系统（A4）负责处理，回合主循环提供快照数据 |
| **胜利/失败在回合中触发** | 立即中断当前阶段，跳转到J胜负判定系统处理 |

## Dependencies 依赖关系

### 上游依赖

回合主循环是Foundation层系统，**无上游依赖**。

### 下游依赖系统

| 系统 | 依赖性质 | 依赖内容 |
|------|---------|---------|
| A2 回合阶段引擎 | 硬依赖 | 当前阶段编号、阶段切换事件 |
| A4 存档系统 | 硬依赖 | 每回合快照数据 |
| A5 游戏时钟 | 硬依赖 | 回合结束触发器 |
| B1-B5 经济系统 | 软依赖 | 回合结算触发 |
| C1-C5 外交系统 | 软依赖 | 回合外交行动触发 |
| D1-D6 军事系统 | 软依赖 | 回合军事行动触发 |
| E 内部政治系统 | 软依赖 | 回合派系更新触发 |
| F1 情报系统 | 软依赖 | 回合情报刷新触发 |
| G 势力AI | 软依赖 | 回合AI响应触发 |
| J 胜负判定系统 | 软依赖 | 回合胜利检测触发 |
| K UI框架 | 硬依赖 | 阶段UI、点数显示、行动菜单 |

**硬依赖** = 本系统不存在则下游完全无法运行
**软依赖** = 下游可独立运转，但需本系统提供触发信号

## Tuning Knobs 调参旋钮

| 旋钮 | 默认值 | 可调范围 | 调整后果 |
|------|--------|---------|---------|
| 总行动点数 | 11 | 8-15 | 过高→选择恐惧；过低→决策单调 |
| 通用点数 | 2 | 1-4 | 过高→阶段区分模糊；过低→灵活性不足 |
| 军事基础配额 | 3 | 2-5 | 过高→军事霸权；过低→侧翼优势消失 |
| 阶段数 | 6 | 4-8 | 过高→操作过细；过低→决策深度不足 |
| 每回合年数 | 0.5年 | 0.25-1年 | 过高→节奏太快；过低→节奏拖沓 |
| 最大回合数 | 24 | 20-30 | 过高→游戏过长；过低→胜利条件难以达成 |

**默认值已确定，保持推荐值。**

## Acceptance Criteria 验收标准

### 基本流程验证
- [ ] 玩家可以在6个阶段中按顺序执行行动
- [ ] 点数消耗正确：每阶段不超过配额+通用点数
- [ ] 跳阶段功能正常：可跳过中间阶段进入后续阶段
- [ ] 0点数时无法执行消耗点数的行动

### AI响应验证
- [ ] AI在第6阶段执行一次响应（不是实时响应）
- [ ] AI响应时玩家无法干预（信息不对称）

### 回合推进验证
- [ ] 回合结束后游戏时钟前进0.5年
- [ ] 回合数正确递增
- [ ] 第24回合结束后游戏时钟显示2040年

### 下游系统触发验证
- [ ] 每回合触发一次经济结算
- [ ] 每回合触发派系满意度更新
- [ ] 每回合触发AI决策计算
- [ ] 每回合触发胜负条件检测

## Open Questions 待解决问题

**已明确的补充设计点：**

1. **阶段结束机制**
   - 玩家可随时点击"进入下阶段"手动结束当前阶段
   - 如果玩家剩余点数未消耗完毕，直接作废
   - 如果玩家想消耗剩余点数，系统不会自动推进（玩家主动结束）
   - 无自动结束强制机制——玩家可以选择"什么都不做就跳到下一阶段"
